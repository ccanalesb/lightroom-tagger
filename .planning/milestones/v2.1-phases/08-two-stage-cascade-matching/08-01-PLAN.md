<action>
Add a new function `compare_descriptions_batch` immediately after `compare_images_batch` in `vision_client.py` with this signature:

```python
def compare_descriptions_batch(
    client: openai_sdk.OpenAI,
    model: str,
    reference_text: str,
    candidates: list[tuple[int, str]],
    log_callback: LogCallback = None,
    max_tokens: int = 4096,
) -> dict[int, float]:
```

Behavior (mirror `compare_images_batch` line-for-line structure where applicable):

1. If `not candidates`: return `{}`.
2. Build a **text-only** user message: system role = same JSON-only system string as `compare_images_batch` (`"You are a JSON-only API..."`). User content = **only** `{"type": "text", "text": ...}` parts — **no** `_image_url_part`, **no** base64.
3. User prompt text must state **text semantic similarity** between the reference summary and each candidate summary; require output **exactly** `{"results": [{"id": <int>, "confidence": <0-100>}, ...]}` with one entry per candidate id, confidence 0–100.
4. Call `client.chat.completions.create` with `temperature=0.1`, `max_tokens=max_tokens`, same `kwargs` pattern: if `"claude" in model.lower()` then `kwargs["extra_body"] = {"reasoning_effort": "none"}`.
5. Parse response like `compare_images_batch`: strip ```json fences if present, `json.loads`, read `parsed["results"]`, build `dict[int, float]` from each item’s `id` and `confidence`.
6. On `json.JSONDecodeError`, `KeyError`, or `ValueError`: log warning via `log_callback` if set, return `{cid: 0.0 for cid, _ in candidates}`.
7. Wrap `client.chat.completions.create` in `try/except Exception` and re-raise via `_map_openai_error(exc, provider=getattr(client, "_provider_id", None), model=model)` exactly like `compare_images_batch`.
</action>
