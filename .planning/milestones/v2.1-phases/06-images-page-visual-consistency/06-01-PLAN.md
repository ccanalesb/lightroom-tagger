# Plan 01: Badge barrel, wrapped primitives, and PerspectiveBadge

## Objective

Move the base `Badge` into `ui/badges/`, refactor specialized badges to render through `<Badge>` with harmonized tokens, add `PerspectiveBadge` (D-11), export everything from `ui/badges/index.ts`, and repoint all consumers so the public API is a single import path (D-01, D-03, D-04).

## Context

- Base primitive today: `apps/visualizer/frontend/src/components/ui/Badge/Badge.tsx` — variants `default | success | warning | error | accent` with shared `rounded-full border` shell.
- Barrel today: `ui/badges/index.ts` exports `ImageTypeBadge`, `ScorePill`, `StatusBadge`, `VisionBadge` only; `Badge` is imported from `../ui/Badge` across ~15 TSX files plus `FilterChip` via `../../ui/Badge/Badge`.
- `VisionBadge`, `StatusBadge`, `ImageTypeBadge` use raw `<span>` + ad-hoc classes (`visionBadgeClasses`, `statusBadgeClasses`, fixed `text-[10px]` for image type).
- `ScorePill` stays score-specific (rounded-full, `descriptionScoreColor`); re-export from barrel and switch `PrimaryScorePill` to `import { ScorePill } from '../ui/badges'` (D-03).

3. **ImageTypeBadge:** Import `Badge`. For `catalog`: `variant="default"` + `className` including `!text-[10px] !font-medium !px-1.5 !py-0.5` and blue-tint surfaces: e.g. `!bg-blue-100 !text-blue-700 dark:!bg-blue-900/30 dark:!text-blue-100 !border-blue-200 dark:!border-blue-800` (tune to match current CAT/IG distinction). 4. **PerspectiveBadge (new file `apps/visualizer/frontend/src/components/ui/badges/PerspectiveBadge.tsx`):** Export `PerspectiveBadge({ perspectiveSlug, score, displayName, className }: { perspectiveSlug: string; score: number; displayName?: string; className?: string })`. Normalize slug with `perspectiveSlug.toLowerCase().replace(/-/g, '_')`. Map to **full** Tailwind override bundles (UI-SPEC): `street` → `!bg-violet-50 dark:!bg-violet-900/20 !text-violet-900 dark:!text-violet-100 !border-violet-200 dark:!border-violet-800`; `documentary` → amber bundle; `publisher` → rose; `color_theory` or `color-theory` → emerald; **else** `variant="default"` on `<Badge>` with no extra color override. **Label text:** if `displayName` is set (e.g. from `IdentityPerPerspectiveScore.display_name`), use it; else derive from title-cased slug (e.g. `street`→`Street`, `color_theory`→`Color theory`). Format children as **`[Label score]`** with `score.toFixed(score % 1 === 0 ? 0 : 1)`. Use `<Badge variant={mappedVariant} className={...}>` so the root is always `Badge`.

5. Export `PerspectiveBadge` from `ui/badges/index.ts`.

6. **JSDoc (D-04):** Add a file-level or per-export JSDoc block above `Badge`, `VisionBadge`, `StatusBadge`, `ImageTypeBadge`, `PerspectiveBadge`, and `ScorePill` in their respective files: one sentence **when** + one sentence **where** (e.g. VisionBadge: vision comparison results in matching UI; ImageTypeBadge: catalog vs Instagram discriminator in description modals; PerspectiveBadge: dominant identity perspective on image tiles and dashboards).
