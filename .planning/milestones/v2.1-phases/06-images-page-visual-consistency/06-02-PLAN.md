# Plan 02: Instagram adapter + inline chip row (Images tabs)

## Objective

Set `ai_analyzed` on Instagram-shaped `ImageView` rows so `ImageMetadataBadges` (inside `ImageTile`) can show the **AI** chip in line with Catalog’s scannable chip row, dedupe “Described” vs **AI** overlay, and keep Matches/Instagram/Catalog (D-06) using the same `ImageMetadataBadges` + `hidePostedMetadataBadge` rules as Best Photos where applicable (D-05, D-07).

## Context

- `ImageTile` (`apps/visualizer/frontend/src/components/image-view/ImageTile.tsx`) always renders `ImageMetadataBadges` in the body with `hidePostedMetadataBadge` optional; Best Photos uses `overlayBadges={<Badge variant="success">Posted</Badge>}` + `hidePostedMetadataBadge={true}` to avoid duplicate Posted (`BestPhotosGrid.tsx` ~163–168).
- `fromInstagramRow` (`adapters.ts` ~99–111) sets `description_summary` from `description` but **not** `ai_analyzed`, so the AI chip never appears on Instagram list tiles.
- `InstagramTab` (`InstagramTab.tsx` ~197–203) adds overlay badges for Matched/Described using `Badge`—after `ai_analyzed` is true, the **Described** overlay duplicates the **AI** chip in the metadata row (both indicate description content).
