# Plan 03: MatchGroupTile card shell + metadata row (UI-03, D-08–D-11)

## Objective

Rework `MatchGroupTile` so the Matches tab shows one Instagram thumbnail with **Catalog tab–style** card affordance (D-09: same `ImageTile` shell as `variant="grid"` catalog tiles) and a **metadata row** below the image/title: validated groups show the **best candidate’s catalog filename** (D-08); unvalidated groups show **only** “N candidates” in that row (D-10), with validation state via `StatusBadge` or `Badge` from the unified barrel (D-01).

## Context

- `ImageTile` (`image-view/ImageTile.tsx`) applies `rounded-card border border-border bg-bg shadow-card transition-all hover:border-border-strong hover:shadow-deep` on the root — this is the Catalog tile shell to preserve.
- Current `MatchGroupTile` (`MatchGroupTile.tsx`) sets `overlayBadges` to either `MATCH_VALIDATED` (`strings`) or `"{count} candidate(s)"` — **no** second-line metadata; D-08/D-10 require moving candidate/validation information into the **body** and clearing redundant overlay text for unvalidated groups.
- `pickInitialMatch` returns the `Match` used for the Instagram side; catalog filename: `initial.catalog_image?.filename` or `filenameFromKey`-style fallback from `fromMatchSide(..., 'catalog')` (see `adapters.ts` `filenameFromKey` is private — use `catalog_image?.filename` or public fields on `Match`).
