Routes (JSON request/response, use `utils/responses` helpers where applicable):

| Method | Path | Behavior |
|--------|------|----------|
| GET | `/api/perspectives` | Query param `active_only=true` optional. Returns list of dicts with keys exactly: `id`, `slug`, `display_name`, `description`, `active`, `source_filename`, `updated_at` (omit raw `prompt_markdown` from list for payload size **or** include it — **must include** `slug` and `active`). |
| GET | `/api/perspectives/<slug>` | Full row including `prompt_markdown`. 404 if missing. |
| POST | `/api/perspectives` | JSON body required keys: `slug` (regex `^[a-z][a-z0-9_]{0,63}$`), `display_name`, `prompt_markdown`. Optional `description`, `active` (default true). Insert into library DB via new helper `insert_perspective` in `database.py` or inline SQL with parameterized queries. |
| PUT | `/api/perspectives/<slug>` | Update `display_name`, `description`, `prompt_markdown`, `active` (partial update allowed — at least one field). |
| DELETE | `/api/perspectives/<slug>` | Delete row; 404 if missing. |
| POST | `/api/perspectives/<slug>/reset-default` | If `source_filename` set, read `Path(repo_root)/prompts/perspectives/{source_filename}` UTF-8; on success replace `prompt_markdown` and bump `updated_at`. If `source_filename` NULL, try `slug + '.md'`. **If the resolved file does not exist:** respond **HTTP 404** with JSON body `{"error": "no default file"}` (exact `error` string — matches API tests). |

**Library DB path:** use same pattern as other API modules: `os.getenv('LIBRARY_DB')` with fallback to `load_config().db_path`.

**Top-of-file comment block** (must be literal triple-quoted or `#` lines): header line exactly `# image_scores JSON field keys (library DB)` followed by comma-separated list: `image_key, image_type, perspective_slug, score, rationale, model_used, prompt_version, scored_at, is_current, repaired_from_malformed`.

Add `insert_perspective` / `update_perspective` / `delete_perspective` to `lightroom_tagger/core/database.py` if not already present (parameterized SQL only).

<threat_model>
**ASVS L1 — perspectives API and UI**

- **Threat:** Unauthenticated modification of LLM system instructions — today’s visualizer is local-trust; mitigate for L1 by **input validation**: `slug` charset `^[a-z][a-z0-9_]{0,63}$`, reject `prompt_markdown` over **256 KiB** with HTTP 400 and body containing `prompt too large`.
- **Threat:** Path traversal on reset-default — only allow `source_filename` matching `^[a-zA-Z0-9_\\-]+\\.md$`; reject else 400 with `invalid source_filename`.
- **Threat:** SQLite injection — only parameterized queries in DB helpers.
- **XSS:** Markdown is instructions for models, not rendered as HTML in browser except in CodeMirror as plain text — do not use `dangerouslySetInnerHTML` for prompt body.
</threat_model>
