<objective>
Expose **read-only** HTTP APIs for `image_scores` so clients can fetch **current** scores for an image (all perspectives with `is_current=1`) and **full version history** for one perspective (all `prompt_version` rows, ordered newest first). Centralize SQL in **`database.py`** helpers. Register a Flask blueprint at **`/api/scores`**. Add **`ScoresAPI`** TypeScript helpers for Phase 6 UI plans.
</objective>

<action>
Create blueprint **`bp = Blueprint("scores", __name__)`** with:

1. **`GET /api/scores/<path:image_key>`** (register url_prefix in app as `/api/scores` so route is `/<path:image_key>`)

   - Query params: **`image_type`** (default `catalog`), **`perspective_slug`** (required ‚Äî return **400** JSON error if missing or empty).
   - Response: `{ "image_key", "image_type", "perspective_slug", "history": [ ... ] }` using **`list_score_history_for_perspective`**.

4. Use **`@with_db`**, **`jsonify`**, and existing **`utils.responses.error_bad_request`** / **`error_not_found`** patterns.

**404 policy:** If no rows at all for that image_key+type for current endpoint, return **`current: []`** (200) ‚Äî not 404 ‚Äî because the image may exist but be unscored.
