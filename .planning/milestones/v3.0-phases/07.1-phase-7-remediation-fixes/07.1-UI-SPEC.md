# Phase 7.1 - UI Design Contract

> Visual and interaction contract for Phase 7 remediation fixes covering embed-job discoverability, queue navigation affordances, and explicit diagnostics for embedding/path failures.

**Sources:** `07.1-CONTEXT.md` (D-01..D-08), `ROADMAP.md` (Phase 7.1), `REQUIREMENTS.md` (STACK-04, STACK-05, NLS-06 dependency context), existing `ProcessingPage`, `ImageDetailModal`, and `JobQueueTab` UI patterns.

**Out of scope:** New stack management workflows, redesign of Processing IA, and net-new product capabilities outside remediation.

---

## Design System

| Property | Value |
| --- | --- |
| Tool | none |
| Preset | not applicable |
| Component library | Existing in-repo UI primitives (`Button`, `Tabs`, `Card` styles) |
| Icon library | none (text-first affordances) |
| Font | Inter (`tailwind.config.js`) |

No new UI package dependencies are allowed for this remediation phase.

---

## Spacing and Layout Contract

| Token | Value | Usage |
| --- | --- | --- |
| xs | 4px | Inline status text spacing |
| sm | 8px | Action group spacing, compact controls |
| md | 16px | Section content padding, card spacing |
| lg | 24px | Section separation on Processing page |

Layout constraints:
- New remediation UI should fit existing card/surface patterns in Processing and modal sections.
- CTA controls that launch jobs must remain touch-friendly (`min-h-11` equivalent when full-width).
- Job Queue navigation affordance should render adjacent to enqueue success feedback, not detached from the action context.

---

## Typography

Use existing semantic scales only:
- Section labels: `text-sm font-semibold`
- Body/help copy: `text-sm`
- Status/error copy: `text-sm` with semantic color classes

Use only weights 400 and 600 for newly introduced copy.

---

## Color and State Contract

Use semantic Tailwind token classes only:
- Informational: `text-text-secondary`
- Success: `text-success`
- Warning/Error: `text-error`
- Surfaces/borders: `bg-surface`, `border-border`

Do not introduce raw hex values or custom ad-hoc color tokens in remediation changes.

---

## Copywriting Contract

Required user-facing intent (exact string constants can vary, but message intent is fixed):

| Element | Copy intent |
| --- | --- |
| Embed trigger label | Clear direct action to start image embedding job |
| Embed enqueue success | Confirms queueing and offers direct path to Job Queue |
| Embed path failure summary | Explains broad path accessibility issue with actionable guidance |
| Skip summary | Condenses grouped skip reasons (no row, empty path, unresolved/missing, encode failure) |
| Similarity degradation notice | States similarity is unavailable due to missing/failed embeddings and points to embed action |

---

## Interaction Contracts

### Discoverable Embed Trigger (D-01, D-03)
- Processing surfaces must include a first-class entry point for `batch_embed_image` (not only fallback in detail modal).
- Trigger must be visible without first inducing a similarity error state.

### Queue Navigation Affordance (D-02)
- After successful enqueue from any remediation surface, render an immediate "Open Job Queue" action that routes to `/processing?tab=jobs`.
- Avoid copy-only guidance as the sole navigation path.

### Diagnostics Visibility (D-06, D-07)
- When embed runs encounter high path accessibility failure rates or grouped skips, UI surfaces a compact diagnosis summary instead of requiring log inspection.
- Similarity-related surfaces that depend on embeddings must show explicit degraded-state messaging.

---

## Accessibility

- Buttons and links for embed trigger/queue navigation must have clear accessible names.
- Async status updates use `role="status"`; failure notices use `role="alert"`.
- Newly added controls follow existing keyboard/focus patterns from in-repo `Button` and `Tabs` usage.

---

## Component Inventory

| Component/Area | Responsibility |

| `constants/strings.ts` | Centralize remediation messaging strings |
| `JobQueueTab` (read-only integration) | Destination tab for remediation navigation flow |
