  Create **new file** `lightroom_tagger/scripts/benchmark_clip_recall.py`.

  1. **Boilerplate:** Shebang `#!/usr/bin/env python3`, module docstring stating read-only recall check for `clip_top_k=50`. After imports, insert the same `sys.path` pattern as `match_instagram_dump.py` line 13: `sys.path.insert(0, str(Path(__file__).parent.parent.parent))` so running as a file resolves `lightroom_tagger`.

  2. **Imports:** `argparse`, `csv`, `os`, `sys`, `sqlite3`, `Path` from `pathlib`. From `lightroom_tagger.core.clip_similarity` import `get_clip_embedding_blob_for_key`, `shortlist_catalog_candidates_by_clip`. From `lightroom_tagger.core.database` import `catalog_key_is_primary_grid_row`, `get_instagram_dump_media`, `get_rejected_pairs`, `init_database`. From `lightroom_tagger.core.matcher` import `find_candidates_by_date`.

  3. **argparse `main()`:** Arguments:
     - `--db` default string **`library.db`**
     - `--out-dir` optional string (default **`.planning/phases/10-match-02-quantitative-benchmark`** — repo-relative path as default so Wave 2 matches D-11/D-12 locations)
     If `not os.path.exists(args.db)`: print error mentioning path and `sys.exit(1)`.

  4. **Truth set query:** `db = init_database(args.db)` in try/finally `db.close()`. One SQL string exactly:
     `SELECT catalog_key, insta_key FROM matches WHERE validated_at IS NOT NULL`
     Execute and build list of `(validated_catalog_key, insta_key)` rows (column order as written).

  5. **Once per run:** `rejected = get_rejected_pairs(db)` before the pair loop.

  6. **Per pair loop** (for each `(validated_catalog_key, insta_key)`):
     - `dump_media = get_instagram_dump_media(db, insta_key)`. If **`None`**: do **not** emit a D-12 CSV row; increment an internal counter `missing_dump_media` for the markdown funnel only (no LLM, no exception).
     - Else set `media_key = insta_key` (must match `dump_media['media_key']` for filter tuple alignment; use the same rejected filter shape as production):  
       `raw = find_candidates_by_date(db, dump_media, days_before=90)`  
       `date_window_size = len(raw)`  
       `candidates = [c for c in raw if (c.get('key'), media_key) not in rejected]`  
       `candidates = [c for c in candidates if c.get('key') and catalog_key_is_primary_grid_row(db, c['key'])]`  
       `cand_keys = [c['key'] for c in candidates if c.get('key')]`  
       `candidates_after_filters = len(cand_keys)`
     - If **`get_clip_embedding_blob_for_key(db, insta_key) is None`**: append CSV row with `shortlist_size=0`, `shortlist_includes_validated=false` (or `0`), **`status=skipped_no_embedding`**; do **not** call `shortlist_catalog_candidates_by_clip` for this pair (D-05: never infer skip from exceptions around shortlist).
     - Else: `short_keys = shortlist_catalog_candidates_by_clip(db, insta_key, cand_keys, top_k=50)` without any `try`/`except` for `NoClipEmbeddingError` (that exception is not raised by this helper per RESEARCH §1).
     - **Classify `status`:**  
       - If `skipped_no_embedding` already assigned, done for that row.  
       - Else if `validated_catalog_key not in cand_keys`: **`status=filtered_out`**  
       - Else if `validated_catalog_key in short_keys`: **`status=hit`**  
       - Else: **`status=miss`**

  7. **CSV output:** Write **`{out_dir}/10-recall-data.csv`** with UTF-8 newline. Header row **exactly** (single line, comma-separated, no spaces after commas):
     `insta_key,validated_catalog_key,date_window_size,candidates_after_filters,shortlist_size,shortlist_includes_validated,status`
     One data row per pair that had `dump_media is not None` (include skipped_no_embedding rows).

  8. **Markdown output:** Write **`{out_dir}/10-RECALL.md`** containing:
     - Title `# Phase 10 — CLIP shortlist recall (validated pairs)`
     - A **funnel** section with markdown table or bullet list including at minimum these integer counts: `total_validated` (SQL row count), `missing_dump_media` (pairs skipped for CSV because no dump row), embedded count (pairs where blob was not None), `skipped_no_embedding`, `filtered_out`, `hits`, `misses`
     - **Recall %** line: define headline as **`100.0 * hits / (hits + misses)`** when `(hits + misses) > 0`, else state denominator zero explicitly (matches RESEARCH §7 “hits / (hits + misses)” on pairs that reached CLIP with validated key in `cand_keys` — i.e. compute hits/misses only from rows where `status` is `hit` or `miss`; exclude `filtered_out` and `skipped_no_embedding` from numerator/denominator)
     - **Miss table:** markdown table listing every row where `status=miss` (columns: insta_key, validated_catalog_key, shortlist_size, date_window_size, candidates_after_filters)

  9. **Entrypoint:** `def main() -> None:` implements the above; `if __name__ == "__main__": main()`
