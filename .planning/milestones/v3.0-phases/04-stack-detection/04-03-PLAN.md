      In `apps/visualizer/backend/jobs/handlers.py`:
      1. Import `fingerprint_batch_stack_detect` from `.checkpoint` (or `jobs.checkpoint` per existing style).
      2. Add `handle_batch_stack_detect(runner, job_id: str, metadata: dict) -> None` with outer `cancel_scope.install(lambda: runner.is_cancelled(job_id))` delegating to `_handle_batch_stack_detect_inner` (mirror `batch_text_embed`).
      3. **Config resolution:** `cfg = load_config(...)` using the same config path pattern as other handlers. Read `raw = metadata.get("delta_ms")`. **Override** only when **`raw is not None` and `raw != 0`** (document in handler: **`0` is falsy/unset**, same as omitting the key—aligns with CONTEXT D-07 `or`, which would fall through to config for `0`). On override: `resolved_delta_ms = int(raw)` and **require `resolved_delta_ms >= 1`** or call `runner.fail_job(job_id, message, severity='warning')` with message containing substring `delta_ms` and return. Otherwise: `resolved_delta_ms = int(cfg.stack_burst_delta_ms)` (enforce `>= 1` if invalid config is possible).
      4. **Force normalization:** Implement helper `_normalize_stack_detect_force(metadata) -> str` returning `"incremental"` | `"full"` | `"preserve_edited"` per D-05. For `"preserve_edited"` in Phase 4, execute the same DB reset as `"full"` (delete all stacks/members) because `user_modified` is always 0 — document in code comment referencing Phase 7.
      5. **Rebuild:** When `force_mode` is `"full"` or `"preserve_edited"`, inside `library_write`: `DELETE FROM image_stacks` (CASCADE removes members) or equivalent idempotent clear.
      6. **Work list:** Build sorted list `all_keys` of every `images.key` intended for this run. For incremental (`force_mode == "incremental"`), `all_keys` = keys where `key NOT IN (SELECT image_key FROM image_stack_members)`; for full rebuild after delete, all catalog `images.key`. **Burst grouping:** fetch `key`, `date_taken`, `rating` for those keys; in Python parse `date_taken` with `datetime.fromisoformat` after normalizing `Z`→`+00:00` if needed; rows with NULL/empty/unparseable `date_taken` increment `images_skipped_no_date` and are excluded from bursts. Sort surviving rows by `(parsed_timestamp_utc, key)`. Scan groups where consecutive gap in milliseconds is greater than `resolved_delta_ms` starts a new group. Only groups with length greater than or equal to 2 become stacks.
      7. **Incremental limitation (document in handler docstring):** Burst detection runs on the **work-list keys only** (not re-evaluating gaps against already-stacked neighbors). Users can run `force: true` to rebuild globally.
      8. **Representative per burst:** Run one SQL query per burst using the pattern from RESEARCH §7 listing `SELECT i.key FROM images i LEFT JOIN ( SELECT s.image_key AS image_key, AVG(s.score) AS ai_score FROM image_scores s INNER JOIN perspectives p ON p.slug = s.perspective_slug AND p.active = 1 WHERE s.is_current = 1 AND s.image_type = 'catalog' GROUP BY s.image_key ) agg ON agg.image_key = i.key WHERE i.key IN (?,...) ORDER BY (i.rating > 0) DESC, i.rating DESC, COALESCE(agg.ai_score, 0) DESC, i.date_taken DESC, i.key DESC LIMIT 1`. Bind burst keys as tuple. `representative_key` must be non-null (D-03).
      9. **Writes:** For each new stack: `INSERT INTO image_stacks (representative_key, stack_size, user_modified) VALUES (?, ?, 0)`; set `stack_size` to member count; insert one `image_stack_members` row per member including representative; all inside `library_write` per stack or batched batches.
      10. **Checkpoint:** Same structure as `batch_text_embed`: `checkpoint_version` 1, `job_type` `'batch_stack_detect'`, `fingerprint` from `fingerprint_batch_stack_detect(metadata, initial_sorted_keys, resolved_delta_ms=resolved_delta_ms, force_mode=force_mode)`, `processed_image_keys` as list of keys fully processed (define: keys assigned to a stack or skipped as singleton from burst scan — exclude only `images_skipped_no_date` from "processed" if they were never candidates; **must** match a clear rule documented in code). Enforce `_CHECKPOINT_MAX_ENTRIES` (reuse constant from handlers if defined for text embed). On mismatch log substring `checkpoint mismatch`.
      11. **Progress:** `runner.update_progress` with formula analogous to text embed (e.g. 5 + 95 * processed/total).
      12. **Counters:** Maintain integers for result: `stacks_created` (INSERT stacks), `stacks_updated` (UPDATE `image_stacks` — use 0 unless you implement in-place updates), `images_stacked` (INSERT into `image_stack_members` this run), `images_skipped_no_date`, `images_skipped_already_stacked` (for incremental: count keys omitted from work list because already members at start).
      13. **Completion:** Before `complete_job`, per CONTEXT D-09, call `add_job_log(job_id, 'info', f'Stack detection complete: {result}; {images_skipped_no_date} images skipped (no date_taken)')` where `result` is a readable summary of the completion counters (e.g. the same dict passed to `complete_job` or a short string built from it). Then `runner.clear_checkpoint(job_id)` then `runner.complete_job(job_id, { "stacks_created": ..., "stacks_updated": ..., "images_stacked": ..., "images_skipped_no_date": ..., "images_skipped_already_stacked": ... })` with all keys exactly those strings (snake_case as shown).
      14. Add `'batch_stack_detect': handle_batch_stack_detect` to `JOB_HANDLERS` dict at bottom of file.
