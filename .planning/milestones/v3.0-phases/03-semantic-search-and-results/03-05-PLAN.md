# Plan 03-05: POST /api/images/semantic-search — NLS-04 row contract + metadata

## Objective

Expose **`POST /api/images/semantic-search`** alongside **`POST /api/images/nl-search`**, reusing **`_rows_to_catalog_api_images`** for catalog field parity and extending each row with **`score`** (RRF), **`why_matched`** (template from plan 03-04), and **`thumbnail_url`** (`/api/images/catalog/{key}/thumbnail`). In `apps/visualizer/backend/api/images.py`:

1. Import `embed_query_to_vec_blob` from `lightroom_tagger.core.embedding_service`
2. Import `run_semantic_hybrid_search` from `lightroom_tagger.core.semantic_search`
3. Import `query_catalog_images_by_keys` from `lightroom_tagger.core.database`

Add route:

```python
@bp.route('/semantic-search', methods=['POST'])
@with_db
def semantic_search_images(db):
```

**Request JSON:**

- `query` (required): non-empty stripped string; if `len(strip) < 2`, return `error_bad_request('query must be at least 2 characters')` (match Phase 1 short-query rule spirit)
- `limit`, `offset`: optional; pass through `_clamp_pagination` like `nl_search_images`
- `score_perspective`: optional string (same meaning as catalog list); pass to `query_catalog_images_by_keys`

**FTS leg validation:**

- Call `match_str, fts_err = build_description_fts_query(str(query).strip())`
- If `fts_err is not None`: return `error_bad_request(fts_err)`
- If `match_str is None`: return `error_bad_request('query must contain at least one searchable term')` — ensures the FTS branch always has a `MATCH` string (**D-08** participation at SQL layer)

**Execution:**

- `blob = embed_query_to_vec_blob(str(query).strip())`
- `rows, total, meta = run_semantic_hybrid_search(db, user_query=str(query).strip(), fts_match=match_str, query_vec_blob=blob, limit=limit, offset=offset)`

- `keys = [r.image_key for r in rows]`
- `catalog_rows = query_catalog_images_by_keys(db, keys, score_perspective=score_perspective_arg)`
- `images = _rows_to_catalog_api_images(catalog_rows, score_perspective_arg)`

**Merge extras:** zip preserves order — for each `images[i]`, set:

- `images[i]['score'] = rows[i].rrf_score` (float)
- `images[i]['why_matched'] = rows[i].why_matched` (str)
- `images[i]['thumbnail_url'] = f"/api/images/catalog/{rows[i].image_key}/thumbnail"`

**Response JSON (exact keys):**

```python
return jsonify({
    'total': total,
    'images': images,
    'metadata': {
        'missing_embeddings_count': meta.missing_embeddings_count,
        'semantic_index_empty': meta.semantic_index_empty,
        'rrf_k': meta.rrf_k,
        'fts_no_match': meta.fts_no_match,
    },
})
```

**Errors:** wrap unexpected exceptions with `error_server_error` like `nl_search_images`.
