# Plan 03-03: batch_text_embed job — selection, checkpoint, vec0 upsert

## Objective

Register **`batch_text_embed`** as a first-class catalog job (**D-05**): it selects catalog images with usable description text (aligned with FTS document rules), embeds with `lightroom_tagger.core.embedding_service`, upserts into **`image_text_embeddings`**, reports **0–100** progress like `batch_describe`, and persists **`checkpoint_version: 1`** state with **`fingerprint_batch_text_embed`**. This delivers the **NLS-03** “store text vectors” requirement.

1. `def _embeddable_document_sql_fragment() -> str` **or** inline WHERE used twice — embeddable means catalog row where:

```sql
(d.description_search_document IS NOT NULL AND TRIM(d.description_search_document) != '')
OR (TRIM(COALESCE(d.summary, '')) != '')
```

2. `def count_catalog_images_missing_text_embedding(conn: sqlite3.Connection) -> int:`  
   Count rows: `images i` join `image_descriptions d` on `d.image_key=i.key AND d.image_type='catalog'` where embeddable fragment true **and** `NOT EXISTS (SELECT 1 FROM image_text_embeddings e WHERE e.image_key = d.image_key)`.

3. `def list_catalog_keys_needing_text_embedding(conn: sqlite3.Connection, *, months: int | None, year: str | None, min_rating: int | None) -> list[tuple[str, str]]:`  
   Return `[(key, 'catalog'), ...]` sorted by `key` ascending:

- Base: `images i` join `d` as above, embeddable fragment true, **not** in `image_text_embeddings`
- If `months` set: `AND i.date_taken >= date('now', ?)` bind `f'-{months} months'`
- If `year` set: `AND strftime('%Y', i.date_taken) = ?`
- If `min_rating` set: `AND i.rating >= ?`

( mirrors `_select_catalog_keys` window semantics )

3b. `def list_catalog_keys_for_text_embed_force(conn: sqlite3.Connection, *, months: int | None, year: str | None, min_rating: int | None) -> list[tuple[str, str]]:` — **same** SQL as (3) but **remove** the `NOT EXISTS (SELECT 1 FROM image_text_embeddings e ...)` clause so all embeddable keys in the date/rating window are returned.

4. `def upsert_image_text_embedding(conn: sqlite3.Connection, image_key: str, vec_blob: bytes) -> None:`  
   **Must** be called **inside** an existing `library_write(conn)` transaction (do not open `library_write` inside this helper to allow batching):

   - `DELETE FROM image_text_embeddings WHERE image_key = ?`
   - `INSERT INTO image_text_embeddings(embedding, image_key) VALUES (?, ?)` with `(vec_blob, image_key)`

If sqlite-vec on disk rejects column order, adjust to the minimal working form verified in plan 03-06 — default assumption: `(embedding, image_key)`.
