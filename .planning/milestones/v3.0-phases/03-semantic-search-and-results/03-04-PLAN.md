# Plan 03-04: Semantic hybrid search — FTS bm25 ranks + vec KNN + RRF + why_matched

## Objective

Implement a **pure-Python** hybrid retriever (**D-07/D-08**): build an **FTS5** ordered candidate list using **`bm25(image_descriptions_fts) ASC`**, build a **sqlite-vec KNN** list from **`image_text_embeddings`** with **`distance_metric=cosine`**, fuse with **RRF `k=60`**, and assemble **template `why_matched` strings (D-10)** from ranks + cosine distance. Enforce **D-09** when the vec table is non-empty: **final ranked keys must have a vec row** (post-filter drops FTS-only unembedded hits). When **`COUNT(*)` on `image_text_embeddings` is 0**, degrade to **FTS-only** ordering (research case (d)). Expose **`missing_embeddings_count`** using `count_catalog_images_missing_text_embedding` from plan 03-03.

Callers still get `missing_embeddings_count` and `fts_no_match: true` in metadata.
- (Continue only when the early return above does not apply.)
- Build rank maps: `fts_rank[key]=1..n`, `vec_rank[key]=1..n`, `vec_dist[key]=distance`

- If `semantic_index_empty`, **do not** filter (FTS-only degradation)
- **Pagination:** `total = len(ordered)`; slice `ordered[offset:offset+limit]`
- Build `SemanticSearchRow` for each sliced key with `rrf_score=scores[key]` and `why_matched=_why_matched_for_key(..., fts_rank=..., vec_rank=..., distance=...)`

Return `(rows, total, SemanticSearchMeta(...))` with `fts_no_match=False` on all non-early-return paths.
