# Plan 11-03 — SearchPage embed discoverability (D-04)

<objective>
When the chat search API returns `metadata.pin_state === 'inactive'` with `metadata.fallback_reason === 'no_clip_embedding'`, show centralized warning copy, a help line, and **two** `react-router-dom` **`Link`** controls to **`PROCESSING_CATALOG_CACHE_ROUTE`** and **`PROCESSING_JOB_QUEUE_ROUTE`** (labels from `SEARCH_PIN_LINK_CACHE` and `PROCESSING_OPEN_JOB_QUEUE`). **No** inline job enqueue on `SearchPage`. </objective>
