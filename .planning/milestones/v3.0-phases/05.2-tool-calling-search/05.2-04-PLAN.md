# Plan 05.2-04 — Frontend multi-turn history

## Objective
Update `SearchPage.tsx` to:
1. Persist `messages` (including tool_call/tool_result turns) returned from the API back into conversation history — so "show me 5 more" sends the correct prior context
2. Show the `assistant_message` (natural language reply) in the chat thread instead of the current "Found N result(s) (nl_filter)" synthetic message
3. Show a subtle "tool calling" indicator in the chat UI when `search_mode === "tool_calling"`

## Tasks

### Task 1 — Persist tool turns and show assistant_message

In `SearchPage.tsx` `handleSubmit`:

Currently:
```typescript
// After API response:
setMessages(prev => [...prev,
  { role: 'user', content: trimmed },
  { role: 'assistant', content: `Found ${data.total} result(s) (${data.search_mode}).` },
])
```

Change to:
```typescript
const assistantText = data.assistant_message
  ?? `Found ${data.total} result(s) (${data.search_mode}).`

// For the conversation display: only show user + assistant text turns (not tool internals)
setMessages(prev => [...prev,
  { role: 'user', content: trimmed },
  { role: 'assistant', content: assistantText },
])

// For the API history: include all turns (tool_call + tool_result) so model has full context
const fullHistory = data.messages ?? [
  ...prior,
  { role: 'user', content: trimmed },
  { role: 'assistant', content: assistantText },
]
// Store fullHistory for next turn's `messages` payload
setApiHistory(fullHistory)
```

Add a new `apiHistory` state (separate from `messages` display state):
```typescript
const [apiHistory, setApiHistory] = useState<Message[]>([])
```

On `handleClear`, also reset `apiHistory`.

In `handleSubmit`, use `apiHistory` (not `messages`) as the `messages` payload:
```typescript
const data = await ImagesAPI.chatSearch({
  message: trimmed,
  messages: apiHistory,   // ← full history including tool turns
  ...
})
```
