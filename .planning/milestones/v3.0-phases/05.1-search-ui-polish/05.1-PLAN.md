# Phase 05.1 — Search UI Polish

## Goal

Fix the four concrete UX issues in the Search page identified after Phase 5 shipped:
layout density, missing clear action, broken mobile layout, and cut-off provider/model selectors.

## Problems to fix

### 1. Clear chat button
The conversation thread has no way to reset. Add a "Clear" button that wipes `messages` and `currentImages` back to initial state.

**Acceptance:** A "Clear" button is visible when at least one message exists. Clicking it empties the thread and result grid.

### 2. Excessive spacing / clunky layout
The two-column layout has too much padding and empty space. The chat column feels disconnected from the results grid. The overall proportions feel loose.

**Acceptance:**
- Reduce vertical gaps between chat bubbles
- Tighten the form area padding
- Chat and results columns feel visually balanced at 1280px+

### 3. Mobile layout
The page was built desktop-first only. On narrow viewports the layout stacks, but spacing, selector row, and chat input are not considered.

**Acceptance:**
- Below `md` breakpoint: single column, chat on top, results below
- Provider+model selector row wraps gracefully or collapses on small screens
- Chat input and Send button don't overflow viewport on mobile

### 4. Provider/model dropdown cut-off
The `<select>` elements for provider and model are truncating long option text (especially NVIDIA NIM model names). On narrow parent containers the text is clipped.

**Acceptance:**
- Dropdowns show enough text to identify the selected model without tooltip
- Long model names don't cause layout overflow
- Consider max-width + `text-overflow: ellipsis` or a tooltip on hover for very long names

## Files in scope

- `apps/visualizer/frontend/src/pages/SearchPage.tsx` — all four fixes
- `apps/visualizer/frontend/src/services/api.ts` — no changes expected

## Out of scope

- Semantic search results display (score badges, why_matched — Phase 6+)
- New search features
- Backend changes
