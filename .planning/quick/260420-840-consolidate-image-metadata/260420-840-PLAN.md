# PLAN: 260420-840 Consolidate image metadata display

## Overview

The visualizer shows the same photo with inconsistent metadata because list rows are adapted into partial “catalog stubs” that zero scores before `CatalogImageModal`, and Instagram uses a separate modal. This work adds a **single backend detail endpoint** that composes catalog row, description, catalog perspective score, and identity scores using existing DB/service helpers, then introduces a shared **`components/image-view/`** module (`ImageView`, adapters, `ScorePill`, badges, tile, and one **`ImageDetailModal`**) that always re-fetches on open. **One modal shell** branches on `image_type` to render **`CatalogImageDetailSections`** vs **`InstagramImageDetailSections`**, keeping Instagram on the full `ImageView` superset with identity fields empty/null (no score pill on tiles). **Mobile**: full-bleed sheet below `md` with `100dvh`, **`@headlessui/react` `Dialog`** for focus management; scroll lock aligned with existing `Modal` behavior.

## Global constraints (apply to every task)

- **Stubs zeroing scores** — mitigated by detail-on-open only (RESEARCH §6.1); ensure UI never shows loader-set zeros as final.
- **Unposted tile sparsity** — missing AI/posted on tile is OK; detail fills (RESEARCH §6.2).
- **Dual score display on cards** — single primary pill via `primaryScoreSource` (RESEARCH §6.4).
- **Deep link empty stub** — detail fetch mandatory (RESEARCH §6.6).
- **Mobile** — prefer **`100dvh`** over **`100vh`**; focus trap via Headless UI (RESEARCH §7).
- **Instagram identity** — SQL/catalog-only identity join; Instagram detail returns empty identity scores (RESEARCH §3).
