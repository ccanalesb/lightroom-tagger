## 3. Backend detail endpoint design

**Proposal:** `GET /api/images/<string:image_type>/<path:image_key>` (register on existing `images` blueprint beside thumbnails `285–317` in `images.py`) returning one JSON object matching frontend `ImageView` / `fromImageDetail`.

**Fields (superset):** `key`, `image_type` (`catalog` \| `instagram`), `filename`, `date_taken`, `filepath` (catalog); LR: `rating`, `pick`, `color_label`, `keywords`, `title`, `caption`, `copyright`, `width`, `height`; `instagram_posted`; AI: `ai_analyzed`, `description_summary`, `description_best_perspective`, `description_perspectives`; catalog list extras: `catalog_perspective_score`, `catalog_score_perspective` (when applicable); identity: `aggregate_score`, `perspectives_covered`, `per_perspective` (same shape as `IdentityPerPerspectiveScore` in `api.ts` `540–548`).

**Compose (reuse, do not reinvent):**

- **Catalog row:** `get_image(db, key)` — `lightroom_tagger/core/database.py` `811–814`.
- **Description:** `get_image_description` — `database.py` `1725+` (same source as descriptions API used in `CatalogImageModal` `85`).
- **Catalog perspective score (single slug):** same join strategy as `list_catalog_images` via `query_catalog_images` with `score_perspective` — `images.py` `452–467`, `471–483`.
- **Identity aggregate + `per_perspective`:** Same computation as `compute_image_aggregate_scores` / `rank_best_photos` — `lightroom_tagger/core/identity_service.py` `81–174`, `216–243`. Implement **single-key** path by filtering one `image_key` after scores query **or** extract `identity_service.get_identity_scores_for_catalog_key(conn, key)` that reuses `_SCORES_BASE_SQL` (`24–40`) and aggregation rules (`125–146`). Join display names via active perspectives (already in SQL).

**Instagram:** Build from `get_instagram_dump_media` + enrich pattern `_enrich_instagram_media` (`images.py` `44–83`). **Confirm:** `_SCORES_BASE_SQL` restricts `s.image_type = 'catalog'` (`identity_service.py` `38–39`); Instagram dump keys are not aggregated there. Detail response: `aggregate_score` / `per_perspective` **null or empty** unless product later scores IG rows — matches CONTEXT (Instagram tiles: no score pill).

**Edge cases:** No description → `ai_analyzed` false, perspectives null. No scores → identity fields null/empty; modal still shows `ImageScoresPanel` run-scoring (`CatalogImageModal` `328–402`). No widening of list endpoints **required** if every modal open calls detail endpoint (CONTEXT Q2); tiles may stay sparse; **Unposted** / **stubs** no longer need to fake scores for modal correctness.

## 4. Responsive modal pattern

**DESIGN.md** breakpoints: `sm 640`, **`md 768`**, `lg 1024`, `xl 1280` (`161–166`).

**Approach:** One wrapper: `fixed inset-0 z-50 md:flex md:items-center md:justify-center md:p-4`; inner: `w-full h-[100dvh] max-h-[100dvh] md:h-auto md:max-h-[min(90vh,900px)] md:max-w-4xl md:rounded-card`. Use **`min-h-0`** on scroll areas so flex children scroll. Prefer **`dvh`** for mobile Chrome/Safari dynamic toolbar vs `100vh` overflow (`Modal.tsx` uses `max-h-[90vh]` `50` — insufficient for full-bleed sheet).

**Existing `Modal.tsx`:** Centered only (`47–51`); sets `body overflow hidden` (`31–37`). Does **not** implement sheet layout — **extend or replace** for `ImageDetailModal`; do not use as-is for “full screen below md”.

## 5. Badge component strategy

**Recommendation:** Keep `Badge` (`components/ui/Badge/Badge.tsx` `1–29`) as-is for semantic chips; add **`ScorePill`** (e.g. `components/ui/badges/ScorePill.tsx` or next to `Badge`) for the **primary score** with props: `source`, `label`, `value`, optional `variant` for description-score coloring — maps to `descriptionScoreColor` usage in `CatalogImageCard` `55–60`.

**Retire three drift styles:** (1) `bg-accent-light` + `rounded-base` aggregate text (`TopPhotosStrip` `93–95`, `BestPhotosGrid` `172–174`), (2) `rounded-full border` perspectives count (`96–98`, `175–177`), (3) raw `descriptionScoreColor` `<span>` (`CatalogImageCard` `55–60`) — all route through `ScorePill` + `Badge` so DESIGN.md badge spec (`112–116`) stays single source.

## 6. Migration order / pitfalls

1. **`bestPhotoToCatalogStub` / `unpostedRowToCatalogStub`:** Eager stubs feed `CatalogImageModal`; removing zeros **requires** detail fetch on open (CONTEXT) — pass only `{ key, image_type }` into modal or stub with explicit “incomplete” flag so UI never shows fake zeros as real (`TopPhotosStrip` `77`, `UnpostedCatalogPanel` `272`).
2. **Unposted + `CatalogImageCard`:** Card reads fields absent on `UnpostedCatalogItem` (`api.ts` `477–482`); falsy/missing → no AI/posted pills — **expected** for tile until detail loads; avoid implying “not posted to IG” vs “unknown” if stub sets `instagram_posted: false` (`53`) for all unposted rows (semantically correct).
3. **`PostNextSuggestionsPanel` `candidateToCatalogStub`:** Forces `instagram_posted: false` (`44`) even if candidate could be inconsistent — detail endpoint must return DB truth.
4. **Dual score pills on catalog card:** Description-derived vs persisted (`CatalogImageCard` `16–29`, `55–66`) — `primaryScoreSource` must encode **one** primary pill per CONTEXT (catalog path uses catalog score with fallback to description best perspective).
5. **Identity inline table vs modal:** `BestPhotosGrid` `189–215` duplicates future `ImagePerspectiveBreakdown` — risk of three copies (strip, expand, modal); consolidate when modal owns breakdown.
6. **Deep link `catalogStubForDeepLink`:** `CatalogTab` `330–337` opens modal with key-only stub — detail fetch **mandatory** before showing scores/metadata.

## 7. Mobile-specific gotchas

- **100vh / safe area:** iOS Safari address bar — use `100dvh` / `min-h-[100dvh]` for sheet; inner scroll with `overflow-y-auto` and `min-h-0`.
- **Focus trap:** Neither `Modal.tsx` nor `CatalogImageModal` / `ImageDetailsModal` implement focus trap — only Escape (`Modal` `19–28`, `CatalogImageModal` `71–77`, `ImageDetailsModal` `45–51`). Full-screen sheet should add **focus management** (planner: headlessui/dialog or similar) when consolidating.
- **Scroll lock:** `Modal.tsx` sets `document.body.style.overflow = 'hidden'` (`31–37`); `CatalogImageModal` does **not** — body may scroll behind overlay; align `ImageDetailModal` with one approach (prefer shared hook).
