<objective>
Persist an exact comparison-pool snapshot for every Instagram media row that reaches matching logic (including zero-candidate and empty-CLIP outcomes), and record **`source_job_id`** when invoked from the visualizer vision-match job so **`--job-id`** filtering can work offline (**19-CONTEXT.md D-01**, **D-03**, **D-10** partially; full CLI in 19-03).
</objective>

<decision_coverage>
- **D-01 / D-03:** Snapshot **`insert_comparison_pool_snapshot`** called from **`match_dump_media`** on all exit paths that record an attempt (see tasks for exact line-level branches).
- **D-10:** Persist UUID string from **`handle_vision_match`** into **`comparison_pool_snapshots.source_job_id`** via new **`match_dump_media`** parameter **`source_job_id: str | None = None`** (keyword-only, placed after **`batch_progress_callback`** in the signature to avoid breaking positional call sites).
- **D-02, D-04–D-09, D-11–D-14:** Implemented in 19-03/19-04; this plan does not add HTML or reconstruction.
</decision_coverage>

<threat_model>
**ASVS level:** 1  
**Block on severity:** high  

**Assets:** Matching pipeline; job UUIDs stored as opaque text; library DB.

**Threats and mitigations:**
- **T-19-double-write / transaction races:** Call **`insert_comparison_pool_snapshot`** **outside** the existing **`with db:`** block that wraps **`store_match`** / stack apply (pass the same **`db`** connection; the insert function commits internally per 19-01 — serialize snapshot **after** scoring, **before** match `with db:` **or** attempted marking, so ordering is: score → snapshot insert → match transaction OR **`mark_dump_media_attempted`**).
- **T-19-PII in job id:** **`source_job_id`** is stored as passed; no logging of full UUID in new code paths beyond existing job logs (do not add **`print(job_id)`** in **`match_dump_media`**).
- **T-19-cancel mid-flight:** If **`should_cancel()`** triggers **after** scoring, snapshot for that media row must **already** be persisted (insert runs immediately after **`score_candidates_with_vision`** returns, before cancel check).

**Residual:** Snapshot volume — acceptable; matches operational reality of diagnostic capture.
</threat_model>
