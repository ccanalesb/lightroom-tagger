<objective>
Extend **`query_catalog_images`** and **`GET /api/images/catalog`** so users can **restrict** the catalog grid to images with at least **min_score** for a chosen **perspective_slug** (using `image_scores.is_current=1`), and **sort** by that score (asc/desc) with deterministic tie-breaker (`i.key`). Update **`ImagesAPI.listCatalog`** and **`CatalogTab`** controls. Optionally surface the **current score** on each card when a score perspective is active for quick scanning.
</objective>

<tasks>
<task>
<type>implementation</type>
<files>lightroom_tagger/core/database.py</files>
<action>
Extend **`query_catalog_images`** signature with optional kwargs (names at implementer choice, suggested):

- **`score_perspective: str | None`** — `perspectives.slug` to join.
- **`min_score: int | None`** — when set with `score_perspective`, add `AND s.score >= ?` (after join).
- **`sort_by_score: str | None`** — allowed values `'asc' | 'desc'`; when set, **require** `score_perspective` or raise `ValueError` (caller maps to HTTP 400).

**SQL strategy (single query for rows + count):**

- Keep existing `FROM images i LEFT JOIN image_descriptions d ON ...`.
- Add **`LEFT JOIN image_scores s`** ON `s.image_key = i.key AND s.image_type = 'catalog' AND s.perspective_slug = ? AND s.is_current = 1` when `score_perspective` is set (bind slug once; use **literal** `catalog` for image_type in SQL text).
- When **`min_score`** filtering: add clause `s.score IS NOT NULL AND s.score >= ?` so unscored images drop out.
- **Sorting:** default remains `i.date_taken DESC` when `sort_by_score` is None. When `sort_by_score` is `desc`, use `ORDER BY (s.score IS NULL) ASC, s.score DESC, i.key ASC` (SQLite: boolean sort). When `asc`, `ORDER BY (s.score IS NULL) ASC, s.score ASC, i.key ASC` — document that **unscored sort last** in both directions unless product chooses otherwise (match `06-CONTEXT` D-31).

- **`COUNT(*)` query** must use the **same joins and WHERE** as the data query so pagination totals stay correct.

- **SELECT list:** include `s.score AS score_for_sort` (or similar) only when needed; strip before API response or map to `catalog_perspective_score` on row dict.

**Performance:** rely on existing indexes `idx_image_scores_current` / `idx_image_scores_perspective_score` — verify `EXPLAIN QUERY PLAN` in a developer note if useful.
</action>
