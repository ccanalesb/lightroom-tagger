**Type:** modify

<action>
1. Add imports: `useState`, `useCallback` from `react`; `DescriptionsAPI`, `ProvidersAPI`, type `ImageDescription` from `../../services/api`; `DescriptionPanel` from `../DescriptionPanel/DescriptionPanel`; `GenerateButton` from `../ui/description-atoms/GenerateButton`.
2. State: `description: ImageDescription | null` initial `null`; `loadingDesc: boolean`; `generating: boolean`; `descError: string | null`.
3. `useEffect` on `[image.key]`: set loading true, call `DescriptionsAPI.get(image.key)`, on success set `description` from `data.description`, on failure set `descError` message, finally `loadingDesc` false.
4. Below the keywords block (after existing keywords UI — locate the closing of keywords section), add a section titled **AI description** using `text-sm font-medium text-text mb-2`.
5. When `loadingDesc`, show `<p className="text-sm text-text-tertiary">Loading description…</p>`.
6. When `descError`, show `<p className="text-sm text-error">{descError}</p>`.
7. Always render `DescriptionPanel` with `description={description}` and `compact={true}` (panel handles null per existing `DESC_PANEL_NO_DESCRIPTION` behavior).
8. Render `GenerateButton` with `hasDescription={Boolean(description?.summary)}`, `generating={generating}`, `onClick` async: set `generating` true, `descError` null; `const defaults = await ProvidersAPI.getDefaults()`; `const d = defaults.description`; call `DescriptionsAPI.generate(image.key, 'catalog', false, undefined, d?.provider ?? undefined, d?.model ?? undefined)`; on response set `description` to `data.description`; on catch set `descError` to `String(err)`; finally `generating` false.
9. In the header badge row next to existing Posted/rating badges, when `description && (description.summary || description.best_perspective)`, show `<Badge variant="accent">AI</Badge>`.
</action>
