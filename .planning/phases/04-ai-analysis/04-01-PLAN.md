<objective>
Expose durable `image_descriptions` rows for catalog images via `GET /api/images/catalog` (optional summary, best perspective, perspectives JSON) and add an `analyzed` tri-state query filter matching the `posted` pattern, with regression tests.
</objective>

<tasks>

### Task 1: Extend `query_catalog_images` with `analyzed` and description JOIN
**Type:** modify
**Files:** `lightroom_tagger/core/database.py`

<read_first>
- lightroom_tagger/core/database.py (functions `query_catalog_images`, `get_undescribed_catalog_images`, and `CREATE TABLE image_descriptions` schema ~lines 221–232)
- apps/visualizer/backend/api/images.py (`list_catalog_images` and `posted` parsing ~lines 264–296)
</read_first>

<action>
1. Add parameter `analyzed: bool | None = None` to `def query_catalog_images(` immediately after the existing `color_label` parameter (keep order: `posted`, `month`, `keyword`, `min_rating`, `date_from`, `date_to`, `color_label`, `analyzed`, `limit`, `offset`).
2. Replace the body’s single-table pattern with queries that always use alias `i` for `images` and `LEFT JOIN image_descriptions d ON i.key = d.image_key AND d.image_type = 'catalog'`.
3. When `analyzed is True`, append clause `d.image_key IS NOT NULL` to `clauses`. When `analyzed is False`, append `d.image_key IS NULL`.
4. Change `COUNT` to: `SELECT COUNT(*) AS cnt FROM images i LEFT JOIN image_descriptions d ON i.key = d.image_key AND d.image_type = 'catalog' WHERE ...` using the same `clauses` and `bindings`.
5. Change the data `SELECT` to: `SELECT i.*, d.summary AS description_summary, d.best_perspective AS description_best_perspective, d.perspectives AS description_perspectives_json FROM images i LEFT JOIN ...` with the same `WHERE`, `ORDER BY i.date_taken DESC`, `LIMIT`, `OFFSET`.
6. Ensure all non-join filters still reference `i.` where ambiguous (e.g. `i.instagram_posted`, `i.rating`, `i.date_taken` in `clauses` that currently omit prefix — update every clause in this function to use the `i.` prefix for `images` columns).
</action>

- `rg "LEFT JOIN image_descriptions d" lightroom_tagger/core/database.py` matches at least one line inside or adjacent to `query_catalog_images`.
- `rg "d.image_key IS NOT NULL" lightroom_tagger/core/database.py` returns at least one match.

</acceptance_criteria>

### Task 2: Serialize catalog rows with description fields for JSON
**Type:** modify
**Files:** `apps/visualizer/backend/api/images.py`

<read_first>
- apps/visualizer/backend/api/images.py (`list_catalog_images` full function)
- lightroom_tagger/core/database.py (`query_catalog_images` return shape after Task 1)
</read_first>

<action>
1. After the existing `posted_raw` / `posted_filter` block, parse `analyzed` exactly like `posted`: `analyzed_raw = request.args.get('analyzed')`; if `analyzed_raw == 'true'` then `analyzed_filter = True`; elif `analyzed_raw == 'false'` then `analyzed_filter = False`; else `analyzed_filter = None`.
2. Pass `analyzed=analyzed_filter` into `query_catalog_images(...)`.
3. In the `for row in rows:` loop, build `out = dict(row)` then `.pop('description_perspectives_json', None)` only (keep `description_summary` and `description_best_perspective` for the response or pop and reassign — **concrete:** pop all three alias keys: `description_summary`, `description_best_perspective`, `description_perspectives_json`).
4. Set `out['ai_analyzed'] = row.get('description_summary') is not None` (after LEFT JOIN, unmatched rows have NULL for `d.summary`; matched rows always have the column present).
5. If `out['ai_analyzed']` is true: set `out['description_summary'] = row.get('description_summary') or ''`; set `out['description_best_perspective'] = row.get('description_best_perspective') or ''`; parse `row.get('description_perspectives_json')`: if truthy string, `out['description_perspectives'] = json.loads(...)` (dict), else `out['description_perspectives'] = {}`. If false: set `out['description_summary'] = None`, `out['description_best_perspective'] = None`, `out['description_perspectives'] = None` (or omit keys — tests in plan 01 expect explicit keys; use `None` for all three when not analyzed).
6. Add `import json` at top of file if not already present.
</action>
