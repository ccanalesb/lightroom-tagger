# Plan 01: Read-only Lightroom catalog connections and I/O documentation

<objective>
Open every **read** path to the `.lrcat` SQLite file using SQLite URI mode `ro`, and publish a short doc that lists which modules read vs write the catalog so browsing and scans cannot mutate the catalog by accident.
</objective>

<tasks>

## Task 1: Implement read-only `connect_catalog` in `lightroom.reader`

<read_first>
- lightroom_tagger/lightroom/reader.py
- lightroom_tagger/lightroom/writer.py
- .planning/phases/01-catalog-management/01-CONTEXT.md
- .planning/phases/01-catalog-management/01-RESEARCH.md
</read_first>

<action>
In `lightroom_tagger/lightroom/reader.py`, replace the body of `connect_catalog(catalog_path: str) -> sqlite3.Connection` so that when environment variable `LIGHTRoom_CATALOG_READONLY_URI` is **unset** or any value other than the literal `0`, the connection is opened with:

- `path_obj = Path(catalog_path).expanduser().resolve()`
- `db_uri = path_obj.as_uri() + "?mode=ro"`
- `sqlite3.connect(db_uri, uri=True, timeout=30.0)`

When `LIGHTRoom_CATALOG_READONLY_URI=0`, keep the previous behavior: `sqlite3.connect(catalog_path, timeout=30.0)` (no `uri=True`), then apply the existing `PRAGMA locking_mode=EXCLUSIVE` branch unchanged (still gated by `LIGHTRoom_CATALOG_LOCKING_MODE`).

For the read-only URI branch, apply the same `PRAGMA locking_mode=EXCLUSIVE` when `LIGHTRoom_CATALOG_LOCKING_MODE` is not `NORMAL`, matching current semantics. Set `conn.row_factory = sqlite3.Row` before return in all branches.

Update the module docstring on `connect_catalog` to state explicitly: **writes to the catalog must use `lightroom_tagger/lightroom/writer.py` (or other documented write tools), not this function.**
</action>

<acceptance_criteria>
- `rg -n "mode=ro" lightroom_tagger/lightroom/reader.py` returns at least one line containing `mode=ro`.
- `rg -n "LIGHTRoom_CATALOG_READONLY_URI" lightroom_tagger/lightroom/reader.py` returns at least one line.
- `rg -n "sqlite3.connect\\(catalog_path" lightroom_tagger/lightroom/reader.py` matches only inside the `LIGHTRoom_CATALOG_READONLY_URI=0` fallback branch (not the default path).
</acceptance_criteria>
