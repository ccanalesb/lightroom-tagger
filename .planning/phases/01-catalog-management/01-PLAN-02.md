# Plan 02: Register `.lrcat` via visualizer settings (shared `config.yaml`)

<objective>
Expose the same `catalog_path` as `lightroom_tagger.core.config.load_config("config.yaml")` through authenticated local-trust Flask endpoints and a visualizer UI panel so users can set the active catalog without hand-editing YAML alone.
</objective>

<tasks>

## Task 1: Add YAML update helper for `catalog_path`

<read_first>
- lightroom_tagger/core/config.py
- apps/visualizer/backend/app.py
- .planning/phases/01-catalog-management/01-CONTEXT.md
</read_first>

<action>
In `lightroom_tagger/core/config.py`, add a function `update_config_yaml_catalog_path(config_file: str, catalog_path: str) -> None` with this behavior:

- Resolve `config_file` with `Path(config_file)`; if the file does not exist, create parent directories with `mkdir(parents=True, exist_ok=True)` and start from an empty dict `{}`; otherwise load with `yaml.safe_load` defaulting to `{}` if the file is empty.
- Set key `catalog_path` in the dict to the string value of `catalog_path` (strip surrounding whitespace; if empty after strip, raise `ValueError("catalog_path must be non-empty")`).
- Write back with `open(..., "w")` and `yaml.safe_dump(data, stream, default_flow_style=False, sort_keys=False, allow_unicode=True)` so existing unrelated keys are preserved when the file already exists.

Do not change `load_config` behavior or `Config` field names.
</action>

<acceptance_criteria>
- `rg -n "def update_config_yaml_catalog_path" lightroom_tagger/core/config.py` returns exactly one match.
- `rg -n "yaml.safe_dump" lightroom_tagger/core/config.py` returns at least one match.
- `rg -n "catalog_path must be non-empty" lightroom_tagger/core/config.py` returns at least one match.
</acceptance_criteria>

<action>
Add new module `apps/visualizer/backend/api/lt_config.py` defining `bp = Blueprint('lt_config', __name__)`.

Compute repo root from `api/lt_config.py` with **four** parent segments (not three like `app.py`): `REPO_ROOT = os.path.abspath(os.path.join(os.path.dirname(__file__), "..", "..", "..", ".."))` — verify at runtime that `os.path.isfile(os.path.join(REPO_ROOT, "pyproject.toml"))` is True in a one-line assertion inside module load or document in code comment.

Implement:

- `GET /catalog` (full path `/api/config/catalog` after registration): call `load_config(LT_CONFIG_YAML)` from `lightroom_tagger.core.config`, return JSON `{"catalog_path": <str from config.catalog_path>, "resolved_path": <str from Path(config.catalog_path).expanduser() if set else "">, "exists": <bool os.path.isfile(resolved)>}` using `jsonify`.

- `PUT /catalog`: parse JSON body with key `catalog_path` (required). Reject with 400 JSON `{"error": "..."}` from `utils.responses.error_bad_request` if: missing key, value not a string, path does not end with case-insensitive suffix `.lrcat`, or `os.path.isfile(expanded_resolved_path)` is False.
