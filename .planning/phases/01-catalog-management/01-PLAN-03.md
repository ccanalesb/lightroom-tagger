# Plan 03: SQL-backed catalog listing, pagination, and extended filters

<objective>
Replace `SELECT * FROM images` plus in-memory slicing in `GET /api/images/catalog` with a single parameterized SQL query and `COUNT(*)` over the same predicates, and expose keyword, rating, date range, and color-label filters through the API and Catalog UI while preserving existing `posted` and `month` behavior.
</objective>

<tasks>

## Task 1: Add `query_catalog_images` to `core.database`

<read_first>
- lightroom_tagger/core/database.py
- lightroom_tagger/core/test_database.py
- .planning/phases/01-catalog-management/01-RESEARCH.md
</read_first>

<action>
In `lightroom_tagger/core/database.py`, add a new public function:

```python
def query_catalog_images(
    db: sqlite3.Connection,
    *,
    posted: bool | None = None,
    month: str | None = None,
    keyword: str | None = None,
    min_rating: int | None = None,
    date_from: str | None = None,
    date_to: str | None = None,
    color_label: str | None = None,
    limit: int = 50,
    offset: int = 0,
) -> tuple[list[dict], int]:
```

Implementation rules:

- Build one SQL `WHERE` clause using **AND** semantics: start from `WHERE 1=1`, then add:
  - `instagram_posted = 1` if `posted is True`, `instagram_posted = 0` if `posted is False`.
  - If `month` is a 6-character string matching `[0-9]{6}`: `strftime('%Y%m', date_taken) = ?` with bound value `month`.
  - If `keyword` is a non-empty stripped string: reuse the same four-column `LIKE` pattern as `search_by_keyword` (`keywords`, `filename`, `title`, `description`) with `COLLATE NOCASE` and bound pattern `%{keyword}%` for each OR branch.
  - If `min_rating` is not `None`: `rating >= ?`.
  - If `date_from` is non-empty: `date_taken >= ?`.
  - If `date_to` is non-empty: `date_taken <= ?`.
  - If `color_label` is non-empty: `LOWER(color_label) = LOWER(?)`.

- Run `SELECT COUNT(*) AS cnt FROM images {where}` with the same bindings list for count.

- Run `SELECT * FROM images {where} ORDER BY date_taken DESC LIMIT ? OFFSET ?` appending `limit` and `offset` to bindings.

- Deserialize rows with `_deserialize_row` and return `(rows, total_count)`.

Do not remove existing `search_by_*` functions.
</action>

- `rg -n "def query_catalog_images" lightroom_tagger/core/database.py` returns exactly one match.
- `rg -n "strftime\\('%Y%m', date_taken\\)" lightroom_tagger/core/database.py` returns at least one match inside `query_catalog_images` or its helper.
- `rg -n "ORDER BY date_taken DESC" lightroom_tagger/core/database.py` returns at least one match in the new functionтАЩs query path.
</acceptance_criteria>
