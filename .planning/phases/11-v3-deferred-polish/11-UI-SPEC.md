### Embed job discoverability (pinned similarity / missing CLIP)

**Primary surface:** `SearchPage.tsx` when the API returns `pin_state: 'inactive'` with `fallback_reason: 'no_clip_embedding'`.

| Element | Copy (contract) |
|---------|-----------------|
| Warning line (status) | Keep the sentence pattern: `Similarity pin inactive: {reason}. Results use your full catalog.` — move `Similarity pin inactive:` / `Results use your full catalog.` to `strings.ts`; map `no_clip_embedding` to a **catalog constant** (e.g. `SEARCH_PIN_WARN_NO_CLIP`) instead of the current hardcoded `'CLIP embedding missing for pinned image'`. |
| Help line (new, below warning) | One short line: e.g. **Run “Embed catalog images only” (or Build catalog cache) under Processing → Catalog cache, then try again.** Exact wording in `strings.ts`. |
| Link — Catalog cache | **Open Catalog cache** — `Link` or `RouterLink` to `PROCESSING_CATALOG_CACHE_ROUTE` (`/processing?tab=cache`). |
| Link / button — Job Queue | **Open Job Queue** — reuse `PROCESSING_OPEN_JOB_QUEUE`; navigates to `PROCESSING_JOB_QUEUE_ROUTE`. Prefer **secondary `Button`** after a successful enqueue, not raw text only. |
| Primary CTA (optional inline) | Reuse `PROCESSING_EMBED_CATALOG_START` or **`Run embed job`** if a shorter label is needed for the narrow column — pick one string in `strings.ts` and use it consistently. |

**Behavior contract**

1. When missing CLIP is detected, the results column shows warning + help + **Open Catalog cache** link (always).
2. Optionally (recommended): **Start embed** action that calls `JobsAPI.create('batch_embed_image', { image_type: 'catalog' })` (or equivalent already used elsewhere); on success, show reuse of `PROCESSING_EMBED_CATALOG_QUEUED` pattern + **Open Job Queue** button (mirror `CatalogCacheTab` success row).
3. Do not regress non–`no_clip_embedding` pin warnings; they keep text-only treatment unless copy is also centralized.

**Secondary surface (todo item 5):** Add a **short troubleshooting** paragraph on `CatalogCacheTab` (below stats or in the cache-location footnote area) explaining that **network share / NAS paths** must be mounted and readable by the backend host, or embed jobs may skip most images — single `strings.ts` constant (≤ ~40 words). No new modal.

**Backend-only** items (preflight, grouped skip counts in job result) are not layout-bearing for this contract; UI consumption goes through existing `JobDetailModal` / result shapes once available.

## Phase 11 — Interaction & accessibility contract

### `AdvancedOptions` disclosure (`matching/AdvancedOptions.tsx`)

**IN-08-01 / Phase 8 polish**

1. The toggle control is the **first `<button>`** (arrow + `ADVANCED_OPTIONS_TITLE`).
2. Set **`aria-expanded={isOpen}`** on that button (boolean).
3. Set **`aria-controls`** to the id of the expandable panel, and **`id`** on the panel wrapper (e.g. `advanced-options-panel`) so assistive tech associates toggle and region.
4. Optionally set **`type="button"`** on the disclosure button to avoid accidental form submit if ever nested in a form (defensive; not a visual change).
5. **Do not** change arrow characters, order of controls, or expand/collapse animation semantics beyond the above.

**Out of scope:** `CollapsibleSection` (`processing` pipeline disclosure) does not block Phase 11; consider the same `aria-expanded` / `aria-controls` pattern in a later pass if time allows.

### `useUndoToast` / `UndoToastBar` (`ConfirmUndoAction.tsx`)

**Phase 7 review low #4**

**Bug:** `offerUndo(message)` with **`onUndo` omitted** currently clears the toast immediately (`setHidden`), so users never read the message.

**Contract:**

1. **`offerUndo(message, onUndo?)`** — If `onUndo` is **provided**, behavior is unchanged: show toast with **Undo** control, auto-dismiss after `timeoutMs` (default **8000** = `DEFAULT_UNDO_TIMEOUT_MS`).
2. If `onUndo` is **omitted**, show a **message-only** toast for the **full** `timeoutMs` duration, then dismiss. No Undo button.
3. **Implement `UndoToastBar`** (or parallel branch) so message-only toasts still render: `role="status"`, `aria-live="polite"`, same positioning/z-index as the undo bar.
4. Do not shorten `timeoutMs` for message-only mode.
