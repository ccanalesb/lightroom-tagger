# Plan 03-02 — Config-driven Instagram keyword for Lightroom writeback

## Goal

Use `lightroom_tagger.core.config.Config.instagram_keyword` (default `"Posted"`, overridable via `config.yaml` or `LIGHTRoom_INSTAGRAM_KEYWORD`) when applying keywords in `update_lightroom_from_matches`. Keep current behavior: **auto-match on successful `vision_match` / CLI match still triggers writeback**; validate/reject semantics stay as today (validate audit-only; reject undoes DB match state per existing `reject_match`).

## Verification criteria

- `python -m pytest lightroom_tagger/lightroom/test_writer.py` exits 0.
- `grep -q "get_or_create_keyword(conn," lightroom_tagger/lightroom/writer.py` and the line uses a variable holding the config keyword, not the literal `"Posted"` inside `update_lightroom_from_matches`.

## must_haves

- `update_lightroom_from_matches` never hardcodes the string `"Posted"` for the keyword passed to `get_or_create_keyword`.
- `match_instagram_dump.py` CLI success message references the configured keyword name, not a hardcoded `"Posted"` only.
