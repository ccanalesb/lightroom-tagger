# Plan 03-05 — Matches review UI

## Goal

Replace the placeholder `MatchesTab` with a working list of match groups from `GET /api/images/matches`, using `useMatchGroups`, `MatchingAPI.list`, and `MatchDetailModal` for validate/reject flows.

## Verification criteria

- `grep -q "useMatchGroups" apps/visualizer/frontend/src/components/images/MatchesTab.tsx` exits 0.
- `grep -q "MatchDetailModal" apps/visualizer/frontend/src/components/images/MatchesTab.tsx` exits 0.
- `grep -q "PLACEHOLDER_MATCHES_VIEW" apps/visualizer/frontend/src/components/images/MatchesTab.tsx` exits 1 (string removed from this file).

## must_haves

- Opening a group shows `MatchDetailModal` wired with `MatchingAPI.validate` / `MatchingAPI.reject` (already inside the modal).
- List reflects pagination or “load more” with `MatchingAPI.list(limit, offset)` where `limit` is at least `50` on first fetch.
