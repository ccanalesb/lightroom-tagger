# Plan 03-01 â€” Matches API + single-match job result

## Goal

Expose Instagram-side thumbnails and metadata for dump-backed `insta_key` values in `GET /api/images/matches`, and return `best_score` on `vision_match` job completion when `metadata.media_key` is set (feeds `useSingleMatch` / `ImageDetailsModal`).

## Verification criteria

- `pytest apps/visualizer/backend/tests/test_match_groups.py` exits 0.
- New or extended test module under `apps/visualizer/backend/tests/` exits 0 and asserts dump-backed `instagram_image.key` in match list JSON.
- `rg "best_score" apps/visualizer/backend/jobs/handlers.py` matches inside `handle_vision_match` near `complete_job`.

## must_haves

- Match groups returned by `GET /api/images/matches` include a non-null `instagram_image` for an `insta_key` that exists only in `instagram_dump_media` (not in `instagram_images`).
- Completed single-image `vision_match` jobs include numeric `best_score` in `result` when `metadata["media_key"]` was provided and at least one match was produced.
