# Plan 03-04 — Dump path UI and import trigger

## Goal

On the Processing **Settings** surface (`SettingsTab.tsx`), let the operator set the server-side Instagram dump directory via the new backend endpoints and enqueue an `instagram_import` job with one click, reusing `ConfigAPI`/`JobsAPI` patterns from `CatalogSettingsPanel.tsx`.

## Verification criteria

- `npm run lint` in `apps/visualizer/frontend` exits 0 (or `npx eslint` with project config if that is the CI gate).
- `grep -q "getInstagramDump" apps/visualizer/frontend/src/services/api.ts` exits 0.
- `grep -q "InstagramDumpSettingsPanel" apps/visualizer/frontend/src/components/processing/SettingsTab.tsx` exits 0.

## must_haves

- User can persist a dump directory path through the UI and trigger `POST /api/jobs/` with `type` exactly `instagram_import`.
- UI copy states that the path is on the **server** (not a browser folder picker), matching IG-01 product intent from research.
