# Plan 03-06 — Posted filter, badges, and API contract

## Goal

Lock in IG-06: catalog browse exposes posted vs not-posted filtering and badges driven by `images.instagram_posted`, consistent with `GET /api/images/catalog?posted=true|false` and stats `posted_to_instagram`.

## Verification criteria

- `pytest apps/visualizer/backend/tests/test_images_catalog_api.py` exits 0.
- `grep -q "posted=true" apps/visualizer/backend/tests/test_images_catalog_api.py` exits 0 (appears in a test request URL or query builder).
- `CatalogTab.tsx` still passes `posted:` into `ImagesAPI.listCatalog` when the posted filter is active.

## must_haves

- Backend test proves `instagram_posted=1` rows are returned only for `posted=true` and excluded for `posted=false`.
- Frontend badges remain `Badge variant="success"` with labels `Posted` / `Posted to Instagram` as today (no removal of user-visible posted indicators).
