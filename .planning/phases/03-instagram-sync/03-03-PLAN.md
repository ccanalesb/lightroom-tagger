# Plan 03-03 — Server-side dump path and import job (backend)

## Goal

Persist a configurable **server filesystem path** to the Instagram export dump (not browser upload), expose it via `/api/config` like the catalog path, and run `import_dump` from `lightroom_tagger.scripts.import_instagram_dump` inside a new job type `instagram_import` registered in `JOB_HANDLERS`.

## Verification criteria

- `pytest apps/visualizer/backend/tests/test_lt_config_api.py` exits 0.
- `pytest apps/visualizer/backend/tests/test_jobs_api.py` exits 0.
- `grep -q "'instagram_import'" apps/visualizer/backend/jobs/handlers.py` exits 0 (inside `JOB_HANDLERS`).

## must_haves

- `POST /api/jobs/` with body `{"type":"instagram_import","metadata":{}}` creates a job that the runner dispatches to a handler calling `import_dump`.
- `GET /api/config/instagram-dump` returns JSON with `instagram_dump_path`, `resolved_path`, and `exists` (boolean directory check).
