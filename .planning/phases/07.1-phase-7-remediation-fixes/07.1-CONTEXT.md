# Phase 7.1: Phase 7 remediation fixes - Context

**Gathered:** 2026-04-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Remediate Phase 7 implementation defects that block reliable pin-similarity and stack-adjacent workflows.

This remediation phase is explicitly focused on:
- Missing/poorly discoverable UI trigger path for the image-embedding flow (`batch_embed_image`)
- Filepath handling defects that use raw/original DB paths where resolved/validated paths are required for embedding reliability

Out of scope:
- New product capabilities unrelated to Phase 7 remediation
- Re-architecture of stack detection logic itself
- Full redesign of Processing information architecture

</domain>

<decisions>
## Implementation Decisions

### Embed flow discoverability and trigger UX
- **D-01:** The embed flow must have an explicit in-product trigger in Processing surfaces (not only in the "Visual similarity is unavailable" detail-modal error state). Planner decides exact tab/control, but it must be first-class and discoverable.
- **D-02:** The existing modal CTA in `ImageDetailModal` is kept, but enqueue success must include a direct navigation action to Job Queue (`/processing?tab=jobs`), not only instructional copy text.
- **D-03:** Any no-embedding state related to pin/similarity must give a clear "run embed job" action path and not rely on users inferring where jobs are started.

### Path resolution and cached-path contract for embedding reliability
- **D-04:** `batch_embed_image` must resolve catalog filepaths via shared path-resolution logic before existence checks and before encoding. Raw `images.filepath` values are not a valid encode input contract.
- **D-05:** Embed job startup must run a small path preflight sample and fail fast with one actionable error when path accessibility is broadly broken (for example, NAS mount unavailable), instead of running long with mostly skip logs.
- **D-06:** Embed job result payload must report grouped skip reasons (for example: no row, empty path, unresolved/missing file, encode failure) so UI can explain failures concisely.
- **D-07:** Stack/pin flows that depend on embeddings must surface clear diagnostics when embeddings are missing due to path accessibility failures; degradation must be explicit, not silent.

### Verification expectations for remediation
- **D-08:** Add targeted tests for: (a) new UI trigger and job-queue navigation affordance, (b) embed-path resolution behavior, (c) preflight fail-fast behavior, and (d) grouped skip-reason payload shape.

### Claude's Discretion
- Exact Processing placement for the new embed trigger control (new section vs existing tab subsection)
- Preflight sample size and fail-fast threshold values
- Exact grouped skip-reason keys in backend result payload, as long as they are stable and UI-consumable

### Folded Todos
- `.planning/todos/pending/2026-04-26-fixes-for-embed-job-discoverability-and-path-failures.md` folded into this phase as the remediation anchor.

</decisions>

## Existing Code Insights

### Reusable Assets
- `JobsAPI.create('batch_embed_image', ...)` already exists and can be reused by a new Processing trigger surface.
- `ProcessingPage` already has query-param tab navigation (`?tab=jobs`) for direct Job Queue linking.
- `resolve_filepath()` in `core/database.py` already handles UNC/NAS normalization.
- `description_service` already demonstrates a stable pattern: resolve path -> validate -> optionally use cached derivative.

### Established Patterns
- Job flows are launched from Processing tabs with clear feedback and Job Queue follow-up.
- Error/fallback messaging is centralized in `constants/strings.ts`.
- Backend jobs report structured completion payloads and detailed logs via `add_job_log`.

### Integration Points
- Add a first-class embed trigger to Processing UI and wire a direct Job Queue navigation affordance after enqueue.
- Update `_handle_batch_embed_image_inner` to resolve/validate paths before `encode_images(...)`.
- Extend embed result payload and UI consumption to expose grouped skip reasons and preflight diagnostics.
