<objective>
Add additive SQLite tables `image_scores` and `perspectives` to the library database via `init_database`, with indexes and typed Python accessors so per-perspective numeric scores are first-class queryable columns (not buried only in JSON). Legacy `image_descriptions` remains unchanged; new rows are nullable by absence (LEFT JOIN semantics documented).
</objective>

<type>implementation</type>
<files>lightroom_tagger/core/database.py</files>
<action>
In `init_database` after the existing `CREATE TABLE IF NOT EXISTS image_descriptions` block and its index, add:

1. **`perspectives` table** (runtime registry; seeding happens in plan 05-02):
   - `id INTEGER PRIMARY KEY AUTOINCREMENT`
   - `slug TEXT NOT NULL UNIQUE` (stable key, e.g. `street`, `documentary`, `publisher`, `color_theory`)
   - `display_name TEXT NOT NULL`
   - `description TEXT NOT NULL DEFAULT ''` (short UI blurb)
   - `prompt_markdown TEXT NOT NULL DEFAULT ''`
   - `active INTEGER NOT NULL DEFAULT 1` (SQLite 0/1 boolean)
   - `source_filename TEXT` (nullable; basename of shipped `.md` for reset, e.g. `street.md`)
   - `updated_at TEXT`
   - `created_at TEXT`

2. **`image_scores` table** (normalized history + “current” flag per D-03):
   - `id INTEGER PRIMARY KEY AUTOINCREMENT`
   - `image_key TEXT NOT NULL`
   - `image_type TEXT NOT NULL DEFAULT 'catalog'` (use `'instagram'` for dump keys when needed)
   - `perspective_slug TEXT NOT NULL` (matches `perspectives.slug`)
   - `score INTEGER NOT NULL` with `CHECK (score BETWEEN 1 AND 10)`
   - `rationale TEXT NOT NULL DEFAULT ''`
   - `model_used TEXT NOT NULL DEFAULT ''`
   - `prompt_version TEXT NOT NULL DEFAULT ''`
   - `scored_at TEXT NOT NULL` (ISO-8601 string, same pattern as `described_at`)
   - `is_current INTEGER NOT NULL DEFAULT 1` (1 = latest row for this image_key+image_type+perspective_slug)
   - `repaired_from_malformed INTEGER NOT NULL DEFAULT 0` (0/1 audit flag for SCORE-07 pipeline)
   - Unique constraint: `UNIQUE(image_key, image_type, perspective_slug, prompt_version)` named `uq_image_scores_versioned`

3. **Indexes** (for Phase 6 filter/sort):
   - `CREATE INDEX IF NOT EXISTS idx_image_scores_perspective_score ON image_scores(perspective_slug, score)`
   - `CREATE INDEX IF NOT EXISTS idx_image_scores_image ON image_scores(image_key, image_type)`
   - `CREATE INDEX IF NOT EXISTS idx_image_scores_current ON image_scores(image_key, image_type, perspective_slug, is_current)`

4. **Python helpers** in the same module (typed, `disallow_untyped_defs`-friendly):
   - `def list_perspectives(conn: sqlite3.Connection, *, active_only: bool = False) -> list[dict]:` — returns all columns as dicts ordered by `slug`.
   - `def get_perspective_by_slug(conn: sqlite3.Connection, slug: str) -> dict | None:`
   - `def insert_image_score(conn: sqlite3.Connection, row: dict) -> int:` — `INSERT`; returns lastrowid; caller manages transaction/`is_current` updates.
   - `def supersede_previous_current_scores(conn: sqlite3.Connection, image_key: str, image_type: str, perspective_slug: str, new_prompt_version: str) -> None:` — sets `is_current=0` for rows matching key+type+slug where `prompt_version != new_prompt_version` (document in docstring).
   - `def get_current_scores_for_image(conn: sqlite3.Connection, image_key: str, image_type: str = 'catalog') -> list[dict]:` — `WHERE is_current=1`.

5. **Module docstring appendix** (single contiguous block at end of module docstring OR immediately above the new helpers): literal heading `## Queryable score fields (image_scores)` followed by a bullet list of every column name and one-line meaning, stating explicitly that missing rows mean “not scored yet” for LEFT JOINs.
</action>

<threat_model>
**ASVS L1 — structured scoring foundation (schema only)**

- **Asset:** Library SQLite files on disk; schema must not weaken integrity of existing tables.
- **Threat:** SQL injection via new helpers — mitigate by using **only** parameterized queries (`?` placeholders) in all new functions; no string interpolation of user slug into SQL beyond validated call sites (API layer in later plans must validate slug charset).
- **Threat:** Migration corruption — mitigate by `CREATE TABLE IF NOT EXISTS` + additive-only DDL; no `DROP` of legacy tables.
- **Privacy:** Score rows reference `image_key` only; no new PII columns. Access control remains the same as today (local/server deployment); this plan does not expose HTTP endpoints.
</threat_model>
