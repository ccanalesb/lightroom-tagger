<objective>
Ship version-controlled default perspective rubrics as markdown under `prompts/perspectives/`, seed the `perspectives` table when empty (D-06), keep DB as runtime truth after seed (D-07). Extract the shared 1–10 rubric from `DESCRIPTION_PROMPT` into reusable content, add **one new** default perspective `color_theory` (SCORE-06). Build description prompts dynamically from active DB rows with a JSON schema whose `perspectives` object keys are **the selected slugs** (dynamic). Refactor `vision_client.generate_description` to accept an explicit text prompt so callers are not locked to the static monolith.
</objective>
