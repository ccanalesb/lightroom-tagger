<objective>
Let users **identify catalog images not yet posted** (POST-04) using the **`/api/analytics/unposted-catalog`** endpoint from 07-01. Primary UX: an **“Not posted”** section on **`AnalyticsPage`** with filters (**date taken range**, **minimum rating**) and a **paginated grid** reusing the same thumbnail / card interaction patterns as the Catalog tab on **`ImagesPage`** (click-through to existing catalog modal or detail flow). Secondary: ensure **Images → Catalog** with `posted=false` remains coherent (copy link or shared filter chip pointing to `/analytics` optional — do not break existing `ImagesAPI.listCatalog` behavior).
</objective>
