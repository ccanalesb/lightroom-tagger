## Spacing Scale

Declared values (multiples of 4). **Use Tailwind numeric spacing only** (e.g. `gap-3`, `space-y-3`, `p-4`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight inline gaps (e.g. `gap-1` in overlay stack) |
| sm | 8px | `gap-2` between related chips/badges |
| md | 16px | Default section and card padding (`p-4`, `space-y-4`) |
| lg | 24px | `space-y-6` between major stack blocks when needed |
| xl | 32px | `space-y-8` page vertical rhythm (Identity/Insights pages) |
| 2xl | 48px | (Reserved — use sparingly) |
| 3xl | 64px | (Reserved) |

**Exceptions:** `ImageTile` thumbnail overlay: `right-2 top-2` (8px) for posted badge (existing pattern). Tab nav uses `px-4 py-2.5` per existing `Tabs` (10px vertical — acceptable; do not change for this phase).

## Color

**Contract:** All UI colors reference CSS variables in `index.css` (light and dark). Use Tailwind semantic classes (`bg-bg`, `text-text`, `text-text-secondary`, `border-border`, `text-accent`, `text-success`, `text-error`) — not raw hex in components.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--color-background)` (`bg-bg`) | Page background |
| Secondary (30%) | `var(--color-surface)` + borders (`bg-surface`, `border-border`) | Cards, charts, image tile surfaces |
| Accent (10%) | `var(--color-accent)` | Active **Insights Top Photos** tab indicator (3px bottom border + label), `focus:ring-accent` on focusable tiles, links |
| Destructive | `var(--color-error)` | Fetch/error messages only (`text-error`, `role="alert"`) |

**Accent reserved for:** Active tab state in the Top Photos control, focus rings on interactive tiles/buttons, and existing app patterns that already use `accent` (e.g. primary links). **Not** for the Posted badge (that uses `success`).

**Posted badge (IDENT-04):** `Badge` `variant="success"` (maps to `text-success` + success-tinted surfaces) — same semantic as `ImageMetadataBadges` “Posted”.

**Light ref (readability only; components must stay theme-token driven):** background ~`#fff` / dark `#191919`, accent ~`#0075de` / dark `#4A9EFF`, success ~`#1aae39` / dark `#3db39a`.

| Identity — Post next section intro | **Unposted candidates the server flags as worth sharing next, with a short reason on each row.** (1–2 sentences; e.g. `IDENTITY_INTRO_POST_NEXT`.) |

| Destructive confirmation | *None* — this phase has no delete/destructive actions. |

## Implementation contracts (downstream)

### IDENT-04 — BestPhotosGrid

- For each `IdentityBestPhotoItem` with `instagram_posted === true`, pass `overlayBadges={<Badge variant="success">Posted</Badge>}` to `ImageTile`. Unposted: omit `overlayBadges` (absence = not posted) — D-01.
- Overlay position is fixed by `ImageTile` (`absolute right-2 top-2`, column `gap-1`).

### IDENT-05 — IdentityPage

- Section order: `StyleFingerprintPanel` → `BestPhotosGrid` → `PostNextSuggestionsPanel` — D-02.
- Immediately under each section’s `h2` with `className="text-card-title text-text"`, add one `<p className="text-sm text-text-secondary">` with the copy from **Copywriting Contract** above — D-03.
- **No** extra card style differentiation between Best vs Post Next beyond existing grid vs list (D-04).

### DASH-02 + DASH-03 — Top scored photos (Dashboard)

- Replace single `getBestPhotos({ limit: 8 })` usage with a **tab strip: Unposted | Posted | All**, default **Unposted** — D-06.
- Fetches: `getBestPhotos({ limit: 8, posted: false })` · `getBestPhotos({ limit: 8, posted: true })` · `getBestPhotos({ limit: 8 })` (omit `posted` for All). All tabs use **8** items unless product later changes (discretion: 8/12 is planner scope).
- **Visual parity:** tab list matches existing `components/ui/Tabs` nav: `border-b border-border`, buttons `px-4 py-2.5 text-sm font-semibold`, active `border-b-[3px] border-accent text-accent`, inactive `border-transparent text-text opacity-60` with hover per `Tabs.tsx`. Content is the existing horizontal `ImageTile` strip pattern (`-mx-1 flex gap-3 overflow-x-auto` from `TopPhotosStrip`). Implementation may use controlled `Tabs` with three tab contents **or** local `useState` + the same classnames — no visual drift.
- **Accessibility:** at minimum `role="tablist"` / `role="tab"` / `aria-selected` on the three tab buttons and one associated tabpanel (or a single `role="region"` with `aria-label` = “Top scored photos” if structure differs). **Do not** use `FilterBar` for this control (user chose tabs over filter chips) — D-06.
- **Loading/empty/error:** per active tab (fetch on tab change or parallel prefetch — planner/executor); each tab shows loading/empty/error states consistent with current `TopPhotosStrip`.
