# Plan 01: Posted filter for best-photos (backend + API client)

## Objective

Implement a real `posted` filter end-to-end so ranked â€śbest photosâ€ť can be constrained to Instagram-posted vs not-posted catalog images before pagination. This unblocks correct semantics for Dashboard Top Photos tabs (DASH-02/DASH-03). The checked-in code does **not** support this today despite prior CONTEXT assumptions.

## Context

- `rank_best_photos` in `identity_service.py` builds an enriched list with `instagram_posted` from `_image_meta_map` but never filters by it.
- `GET /api/identity/best-photos` does not read a `posted` query param.
- `IdentityAPI.getBestPhotos` in `api.ts` does not accept or send `posted`.
- Filtering must happen **before** `offset`/`limit` slicing so totals and page contents reflect true top-N for that posted state.
