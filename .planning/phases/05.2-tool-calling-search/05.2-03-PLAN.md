# Plan 05.2-03 — API wiring

## Objective
Update `POST /api/images/chat-search` to branch on provider capability: when the selected model supports tool calling, use `run_tool_calling_search`; otherwise fall back to the existing `run_nl_catalog_filter_llm_multi_turn` path. Return tool turns in the response so the frontend can persist them.

## Tasks

### Task 1 — Update `POST /api/images/chat-search` in `apps/visualizer/backend/api/images.py`

Current flow: always calls `run_nl_catalog_filter_llm_multi_turn` → parses JSON filters → runs query.

New flow:
1. Determine if the selected provider supports tool calling:
   ```python
   registry = _get_registry()  # or ProviderRegistry()
   provider_config = registry.get_provider(resolved_provider or desc_defaults.get("provider"))
   use_tool_calling = bool(provider_config.get("tool_calling", False)) if provider_config else False
   ```

2. **Tool-calling path** (when `use_tool_calling` is True):
   ```python
   from lightroom_tagger.core.nl_catalog_search import run_tool_calling_search
   
   assistant_text, updated_messages = run_tool_calling_search(
       turns_for_llm,
       provider_id=body.get("provider_id"),
       model=body.get("model"),
       db=db,
   )
   
   # Extract images from the last tool result in updated_messages
   images, total = _extract_images_from_tool_messages(updated_messages, db)
   
   return jsonify({
       "images": images,
       "total": total,
       "search_mode": "tool_calling",
       "filters": None,
       "metadata": None,
       "messages": updated_messages,  # includes tool_call + tool_result turns
       "assistant_message": assistant_text,
   })
   ```

3. **Fallback path** (when `use_tool_calling` is False): existing logic unchanged.

Add helper `_extract_images_from_tool_messages(messages, db)`:
- Find the last `tool` role message in messages
- Parse its `content` JSON
- Return `(images_list, total)` from the tool result
- If no tool messages found, return `([], 0)`

Also add `_get_registry()` helper if not already present (returns `ProviderRegistry()` singleton).

The images returned should use the existing `_rows_to_catalog_api_images` format (with thumbnail_url etc.) — re-fetch from DB by key if needed, or adapt `_rows_to_tool_result` to include thumbnail_url.
