<objective>
Before any `.lrcat` mutation, refuse writes when Lightroom holds the catalog (lock files), then create a timestamped `shutil.copy2` backup in the catalog directory with rotation capped at two backup files, matching D-03–D-09. Centralizing this in `writer.py` ensures future catalog write paths inherit the same safety without duplicating handler logic.
</objective>

### Task 2: Timestamped backup with max-two rotation
**Type:** modify
**Files:** lightroom_tagger/lightroom/writer.py

<read_first>
- lightroom_tagger/lightroom/writer.py
- lightroom_tagger/lightroom/cleanup_wrong_links.py
- .planning/phases/02-jobs-system-reliability/02-CONTEXT.md
</read_first>

<action>
1. Add `def backup_catalog_if_needed(catalog_path: str, *, max_backups: int = 2) -> str:`:
   - Let `cat = Path(catalog_path)`; `parent = cat.parent`; `pattern = f"{cat.name}.backup-*"`.
   - List `existing = sorted(parent.glob(pattern), key=lambda x: x.stat().st_mtime)` (oldest first).
   - **While** `len(existing) >= max_backups`, delete `existing.pop(0)` using `path.unlink(missing_ok=True)` and refresh list OR re-glob after each delete; end state must allow adding one new backup so **at most `max_backups` files** match `pattern` after the function completes.
   - Build `ts = datetime.now().strftime("%Y-%m-%dT%H-%M-%S")` (hyphens, no colons).
   - `dest = parent / f"{cat.name}.backup-{ts}"`.
   - `shutil.copy2(catalog_path, dest)`.
   - Log: `logger.info("Catalog backup created: %s", dest)` — exact prefix substring `Catalog backup created:` must appear.
   - Return `str(dest)`.
2. Document in a one-line comment above `backup_catalog_if_needed` that backups are **per write call** (caller `update_lightroom_from_matches` runs once per job invocation).
</action>

- `rg "\.backup-" lightroom_tagger/lightroom/writer.py` matches.
</acceptance_criteria>

<threat_model>
| Threat | Severity | Mitigation |

| Disk exhaustion from repeated backups | Medium | `max_backups=2` rotation; single copy per `update_lightroom_from_matches` invocation. |
| Path traversal via crafted catalog_path | Low | Caller supplies configured path only; no new user input concatenation beyond `Path` operations on that path — keep validation unchanged from Phase 1. |
