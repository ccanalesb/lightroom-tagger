<action>
Create `apps/visualizer/frontend/src/pages/SearchPage.tsx`:

1. Types:

```ts
type Message = {
  role: 'user' | 'assistant'
  content: string
  images?: CatalogImage[]
  search_mode?: string
}
```

2. State: `messages: Message[]`, `input: string`, `status: 'idle' | 'loading' | 'error'`, `errorText: string | null`, `currentImages: CatalogImage[]`, `selectedImage: { key: string; initial?: CatalogImage } | null` (mirror `CatalogTab` selected shape).

3. Layout wrapper inside page shell:

```tsx
<main className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-6">
  <div className="flex flex-col md:flex-row gap-6 h-full min-h-[60vh]">
```

Left column: `className="w-full md:w-2/5 flex flex-col min-h-[320px]"` — scrollable message area (`flex-1 overflow-y-auto space-y-3`), user bubbles `text-right`, assistant `text-left`, subtle bubble styling using existing `bg-surface` / `border-border` tokens. Bottom: `<form>` with labeled text input (`<label className="sr-only">` or visible label) + submit button; on submit prevent default. When `status === 'error'` and `errorText` is set, render below the input (still in the left panel): `<p role="alert" className="text-red-500 text-sm mt-2">{errorText}</p>`.

Right column: `className="w-full md:w-3/5"` — when `status === 'loading'`, show `<SkeletonGrid count={12} />`; when `currentImages.length === 0` and `messages.length === 0`, centered copy **exactly**: `Ask about your photos...`; when `status === 'idle'` and `messages.length > 0` and `currentImages.length === 0`, show `<p className="text-center text-gray-500 mt-8">No matches found. Try a different query.</p>`; when results exist, render:

```tsx
<TileGrid>
  {currentImages.map((image) => (
    <ImageTile
      key={image.id != null ? String(image.id) : image.key}
      image={fromCatalogListRow(image)}
      variant="grid"
      primaryScoreSource="catalog"
      onClick={() => setSelectedImage({ key: image.key, initial: image })}
    />
  ))}
</TileGrid>
```

Wire `ImageDetailModal` like `CatalogTab`: `open={!!selectedImage}`, `imageKey={selectedImage?.key}`, `initialImage={selectedImage?.initial ? fromCatalogListRow(selectedImage.initial) : undefined}`, `onClose={() => setSelectedImage(null)}`.

4. Submit handler:

- If `!input.trim()` or `status === 'loading'`, return.
- `setStatus('loading')`, clear error.
- Append `user` message to `messages`.
- Build `prior` for API: map existing `messages` (before this turn’s assistant reply) to `{ role, content }[]` with only `user`/`assistant` roles.
- `await ImagesAPI.chatSearch({ message: input.trim(), messages: prior, limit: 50 })`.
- On success: `setCurrentImages(data.images)`; append assistant message with content like `Found ${data.total} result(s) (${data.search_mode}).` (adjust grammar); store optional `search_mode` on the `Message`.
- On failure: `setStatus('error')`, `setErrorText(err.message)` (from `Error`).
- Finally `setStatus('idle')`, clear input on success only (or always clear — prefer clear on success).

5. Disable input + button while `status === 'loading'`.

6. Export `export function SearchPage()`.
</action>
