<objective>
Ship a dedicated **Identity** experience that consumes **`/api/identity`** endpoints: a **sortable / paginated Best Photos grid** (aggregate score + perspective breakdown on demand), a **Style Fingerprint** visualization (**Recharts** radar or bar chart for per-perspective means + supporting distribution/token lists), and a **What to Post Next** panel listing suggestions with **reason chips** and **navigation into the catalog** (reuse existing image modal / catalog routes). Match **AnalyticsPage** patterns for loading, errors, empty states, and accessibility. **No new NLP or embedding UI** — display server-provided numbers, tokens, and reasons only.
</objective>

<threat_model>
**Client-side exposure**

- **Threat:** XSS from API text fields (`rationale`, reasons) — React text nodes escape by default; **do not** `dangerouslySetInnerHTML` on model text.
- **Threat:** Leaking sensitive filenames in screenshots — out of scope; same as catalog UI.
- **Privacy:** Identity page may surface **aggregate behavioral patterns** — acceptable under local/trusted deployment; no additional telemetry required.
</threat_model>
