# Plan 03-01: sqlite-vec extension load + image_text_embeddings vec0 migration

## Objective

Pin and install **sqlite-vec 0.1.9** and **sentence-transformers** (with its transitive stack) so the library DB can store 768-dimensional **cosine** text embeddings in the same SQLite file as the catalog. Load the extension on every `init_database` connection and add an idempotent migration (PRAGMA `user_version` **3 → 4**) that creates the **`image_text_embeddings`** `vec0` virtual table. This is the storage foundation for **NLS-03** (`batch_text_embed` + hybrid search).

2. Add a **module-level** helper:

```python
def _ensure_sqlite_vec_loaded(conn: sqlite3.Connection) -> None:
    conn.enable_load_extension(True)
    sqlite_vec.load(conn)
    conn.enable_load_extension(False)
```

3. At the start of `init_database`, **immediately after** `conn = sqlite3.connect(db_path, check_same_thread=False)` and **after** `conn.row_factory = _dict_factory`, call `_ensure_sqlite_vec_loaded(conn)`.

4. If `sqlite_vec.load` raises (missing binary, unsupported Python SQLite build without `enable_load_extension`), let the exception propagate — do not swallow it.

No changes to WAL / busy_timeout pragmas beyond their existing order; extension load must occur before any DDL that uses `vec0` (migration runs later in the same function).
</action>

<action>
In `lightroom_tagger/core/database.py`:

1. Add a new function `_migrate_image_text_embeddings_vec0(conn: sqlite3.Connection) -> None` **placed immediately after** `_migrate_image_descriptions_fts` (same file region as other migrations).

2. Migration logic (mirror FTS migration structure):
   - Read `current_uv = int(conn.execute("PRAGMA user_version").fetchone()["user_version"])`.
   - If `current_uv >= 4`, return immediately.
   - Inside `try`:
     - `conn.execute("DROP TABLE IF EXISTS image_text_embeddings")` (idempotent dev resets; matches FTS migration style).
     - `conn.execute` the following DDL **exactly** (whitespace may be formatted by Black):

```sql
CREATE VIRTUAL TABLE image_text_embeddings USING vec0(
  embedding float[768] distance_metric=cosine,
  image_key TEXT
);
```

   - `conn.execute("PRAGMA user_version = 4")`
   - On `sqlite3.OperationalError`, `return` **without** bumping `user_version` (same pattern as `_migrate_image_descriptions_fts`).

3. In `init_database`, call `_migrate_image_text_embeddings_vec0(conn)` **after** `_migrate_image_descriptions_fts(conn)` and **before** `seed_perspectives_from_prompts_dir(conn)`.

4. Do **not** backfill embeddings in this plan — empty table until `batch_text_embed` runs.
</action>
