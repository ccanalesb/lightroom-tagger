# Phase 05.1 — Search UI Polish — COMPLETE

## What was built

All four issues fixed in a single pass on `SearchPage.tsx`.

### 1. Clear button
- Renders above the thread only when `messages.length > 0`
- Resets `messages`, `currentImages`, `errorText`, `status`, and `input` in one click
- Disappears on empty state (no visual clutter on load)

### 2. Tighter layout
- Header margin: `mb-6` → `mb-3`, subtitle font size `text-sm`
- Main flex gap: `gap-6` → `gap-4`
- Chat bubbles: `space-y-3` → `space-y-1.5`, `py-2` → `py-1.5`
- Form gap: `gap-2` → `gap-1.5`
- Assistant bubble now uses `text-text-secondary` to visually distinguish from user
- Send button shows `…` while loading instead of staying static
- Empty states vertically centered in results column

### 3. Mobile layout
- Selector row: `flex gap-2` → `flex flex-wrap gap-1.5` — wraps gracefully on narrow viewports
- Each selector group has `min-w-[120/140px] flex-1` so they share space when available and stack when not
- Input row uses `items-center` for consistent alignment
- Thread auto-scrolls to bottom on new messages via `useRef`

### 4. Dropdown cut-off
- All `<select>` elements get `truncate` class (CSS `text-overflow: ellipsis`)
- `title` attribute on `<select>` shows full selected value on hover
- `title` attribute on each `<option>` shows full model name on hover
- Long NVIDIA NIM names (`nvidia/llama-3.1-nemotron-ultra-253b-v1` etc.) are readable on hover without layout overflow
