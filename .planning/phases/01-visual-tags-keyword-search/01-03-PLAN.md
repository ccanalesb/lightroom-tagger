## Objective

Expose NLS-02 end-to-end: pure `build_description_fts_query(raw: str | None) -> tuple[str | None, str | None]` where the first value is the FTS `MATCH` string (tokens AND-joined after sanitization) or `None` when no FTS filter should apply, and the second is `None` or a short error code/message for the API layer; extend `query_catalog_images(..., description_search: str | None = None)` to constrain catalog rows to `image_type='catalog'` FTS matches only; wire `GET /api/images/catalog?description_search=` with validation (trimmed empty → omit filter; trimmed length 1 → 400 with body `{"error": "description_search must be at least 2 characters"}`; length ≥ 2 → sanitize); add unit/integration tests including injection-style inputs; add a separate `search` filter descriptor on `CatalogTab` with `paramName: 'description_search'` (D-08/D-09), `ImagesAPI.listCatalog` typing, and RTL test asserting the query param after debounce/commit.

   - Do not embed user substrings into SQL — only return the match string for use as a single bound parameter.

3. Add unit test: `build_description_fts_query("a")` returns `(None, "description_search must be at least 2 characters")`.
