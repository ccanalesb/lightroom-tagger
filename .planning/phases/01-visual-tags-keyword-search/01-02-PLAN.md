## Objective

Extend the single-call describe JSON contract so the model returns top-level `dominant_colors` (array of hex strings), `mood_tags` (array of strings), and `has_repetition` (boolean). Map these in `_store_structured` into `store_image_description` with explicit fallbacks: `dominant_colors` from top-level else `technical.get('dominant_colors')` (list of hex strings, pass through); `mood_tags` always a JSON list — top-level list if present, else if only `technical.get('mood')` is a non-empty string wrap `[that string]`; `has_repetition` coerced to 0/1 or NULL. Update `analyzer.DESCRIPTION_PROMPT` and `_DESCRIPTION_FALLBACK` in `lightroom_tagger/core/analyzer.py` so the documented JSON shape and parse-failure defaults include the same root keys (`dominant_colors`, `mood_tags`, `has_repetition`). Deserialize new JSON columns in `get_image_description`. Extend `_DESC_JSON_COLS` (or equivalent) in `images.py` so API responses return objects not raw strings for `dominant_colors` and `mood_tags`. Update any inline `store_image_description` dict in `match_instagram_dump.py` to omit new keys (NULL-safe) or pass through when available.
