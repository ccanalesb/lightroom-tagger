## Objective

Implement D-18/D-19: user-initiated backfill that re-describes catalog images which already have an `image_descriptions` row but `dominant_colors IS NULL`, using the existing `batch_describe` job. This plan **completes VIS-01** by enabling user-initiated backfill of visual tags for existing described images. Add boolean metadata `backfill_visual_tags` (default `false`) read by selection SQL in the describe handler path; extend `fingerprint_batch_describe` canonical JSON payload with key `"backfill_visual_tags": true|false` so checkpoints invalidate when the mode changes. Surface an Advanced-options checkbox on `AnalyzeTab` labeled **Backfill visual tags (re-describe images with missing color/mood data)** that sets `backfill_visual_tags: true` in the job metadata JSON. Backfill refreshes structured fields and `description_search_document` via `store_image_description`.
