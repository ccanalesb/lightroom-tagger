# Plan 04: Dominant PerspectiveBadge on Best Photos and Dashboard strip

## Objective

Show the top-1 identity perspective (highest `score` in `per_perspective`) on `ImageTile` using the `footer` slot (D-12) in `BestPhotosGrid` (Identity) and `TopPhotosStrip` (Dashboard), reusing `PerspectiveBadge` from `../ui/badges` and `IdentityPerPerspectiveScore` from `services/api` (`perspective_slug`, `display_name`, `score`).

## Context

- `IdentityBestPhotoItem` (`api.ts` ~589–600) has `per_perspective: IdentityPerPerspectiveScore[]` with `perspective_slug`, `display_name`, `score`.
- `fromBestPhotoRow` (`adapters.ts` ~69–81) already maps `identity_per_perspective: row.per_perspective` onto `ImageView`.
- `ImageTile` (`image-view/ImageTile.tsx` ~19, 94–95) supports optional `footer` **after** `ImageMetadataBadges`; it is unused in production today (tests only).
- `BestPhotosGrid` (`136–169`) uses `ImageTile` `variant="compact"` with `overlayBadges` for Posted + `hidePostedMetadataBadge` — the **footer** is still free for `PerspectiveBadge`.
- `TopPhotosStrip` (`45–55`) uses `variant="strip"` and no overlay — add `footer` the same way.
