## Phase 8 ‚Äî Surfaces & interaction contract

### Catalog Cache (`CatalogCacheTab`)

1. **Primary row:** One prominent **Build catalog cache** CTA (`variant="primary"`, `size="lg"`, `fullWidth` or consistent with current primary actions) that starts the full pipeline (D-04). Placed at the top of the action area after the intro/stats block (order: value-first, then chain CTA).
2. **Advanced options:** A single disclosure using **`AdvancedOptions`** (D-05) ‚Äî same toggle affordance (‚ñ∂/‚ñº + title) as `MatchingTab`. Inside: individual `JobsAPI.create` actions for at least:
   - `batch_embed_image` (existing behavior extended for IG; label clarifies catalog + Instagram when backend supports it),
   - `batch_stack_detect`,
   - `batch_catalog_similarity`,
   - `prepare_catalog` (legacy ‚Äî only here, not duplicated at top level).
3. **Remove duplication:** Any standalone "Embed" block that competes with the chain must be **merged** into the Advanced section or de-emphasized so **one** primary story exists: chain first, surgery second.
4. **Progress:** Reuse existing cache stats grid and progress bar; after chain job enqueue, show success line + **Open job queue** secondary action (same pattern as embed success).

### Matching (`MatchingTab`)

1. **CLIP top-k:** A **numeric input** (not slider) between date filter and `AdvancedOptions`, default **50** (D-02). Binds to `vision_match` metadata (e.g. `clip_top_k` ‚Äî exact key is backend discretion; UI sends integer).
2. **Bounds:** **1 ‚â§ k ‚â§ 500**; clamp on blur or validate on submit with the error copy above.
3. **Removal:** The entire **Catalog Discovery Jobs** card (stack detect, catalog similarity, preview) is **removed** from `MatchingTab` (D-06). Optionally keep **only** a short informational line + link: "Stack and similarity jobs run from **Catalog cache**" with `TAB_CATALOG_CACHE` or tab route ‚Äî copy in `strings.ts`.
4. **Metadata:** `vision_match` continues to send existing fields; add top-k. Weight/skip/provider unchanged.

### Observability (user-visible)

- Users verify pipeline behavior in **Job Queue** / **Job detail** (logs), not a new analytics dashboard. No new real-time in-tab chart required for Phase 8.
- If job API exposes progress keys later, the tab may show a one-line **status** readout; optional, not blocking.

### Accessibility

- Top-k field: associated `<label>`, `inputMode="numeric"`, `min`/`max` attributes matching 1‚Äì500, describe purpose in helper text (CLIP shortlist before LLM scoring).
- Advanced toggles: preserve button + expanded region semantics; if `AdvancedOptions` is extended, do not remove keyboard operability of the disclosure control.
