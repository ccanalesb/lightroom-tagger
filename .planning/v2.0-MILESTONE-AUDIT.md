## Integration Issues

### Critical

1. **`batch_score` non-force image selection (Phase 5 → 6)**
   - `handle_batch_score` uses `get_undescribed_catalog_images` / `get_undescribed_instagram_images` when `force` is false
   - These helpers return images **without descriptions**, not images without scores
   - After bulk describe, batch scoring with Force off queues **no images**
   - Affects: SCORE-01, SCORE-04, IDENT-01

### Warning

2. **Identity suggestions `offset` silently dropped (Phase 8)**
   - `/api/identity/suggestions` parses `offset` via `_clamp_pagination` but does not pass it to `suggest_what_to_post_next`
   - Frontend currently only uses `limit`, so latent bug
   - Affects: IDENT-03

3. **Identity aggregation key collision risk (Phase 5 → 8)**
   - `compute_image_aggregate_scores` buckets by `image_key` only, not `(image_key, image_type)`
   - If a catalog key matched an Instagram media_key, scores could merge
   - Unlikely in practice but possible given the data model
   - Affects: IDENT-01, IDENT-02
